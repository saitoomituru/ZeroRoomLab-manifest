## 2. User Gate決定

### 2.1 default

- defaultは`OPEN / gift-commons / non-exclusive`
- 神、作者、実装者、詩人、生活様式、先行思想との関係は多対多であり、参照で対象を取り上げない
- provenance、lineage、structural similarityは、所有権、本人性、権威、順位、許諾を生成しない
- 誰が書いた、思いついた、実装した、観測した、実践したかを、次元付きで記憶できる
- byte、code、algorithm、architecture、philosophy、imagination、poem、faith、
  lifestyle、aestheticを同じ「コピー」の一語へ潰さない

### 2.2 閉じる自由

- private、local-only、commercial、閉鎖World、宗派World、魔王城を作る自由は否定しない
- 閉鎖は選択されたApp、integrator glue、asset、Worldへ局所化する
- 閉鎖状態はcore、framework interface、既存commons、lineage graph、無関係なWorld、
  upstream fork権へ逆流させない
- provider契約、課金、SLA、独自assetの保証は、選択した第三者App／integratorの責務とする
- public設定へcredentialそのものを置かず、外部secret referenceだけを記録する

### 2.3 conflict

- asset競合は、選択されたread／publish／execute routeだけを停止できる
- 局所競合を全World、全repository、全参加者のglobal banへ拡張しない
- unmount、replacement、fork、別World routeを回復経路として残す
- capability／security／物理安全等の別scopeにglobal stopが必要な場合は、
  asset lineage判断ではなく、そのauthorityと根拠を別receiptへ記録する

### 2.4 探索資源

- read、fork、remix、批評、失敗receipt、別World生成をprotocol上の探索UBIとして開く
- 食料、住居、時間、計算資源、hardware等の物質的探索UBIは必要なSupply課題だが、
  このpolicyが供給済みと主張しない
