# ドキュメント整備規約

作成日: 2026-07-13

## 1. 目的

この文書は、ZeroRoomLab-manifest の文書群が増えたときに、配置・命名・更新責任を混乱させないための規約を定める。

## 2. 基本方針

- 文章の内容は AGENTS.md の三層規約に従う。
- 物理的な配置は、運用層・正規化層・ノート層の役割分担に従う。
- 1つの内容を複数の文書へハードコピーしない。
- 更新が必要な箇所は、元の原典を更新し、参照元はリンクだけ差し替える。

## 3. 配置ルール

- [docs/philosophy](../philosophy) には哲学・思想・世界観文書を置く。
- [docs/theory](../theory) には技術・仕様・実装方針文書を置く。
- [docs/projects](../projects) にはプロジェクト・実装計画・実験ログを置く。
- [docs/operations](.) には運用・ガバナンス・転送ルールを置く。
- [note](../../note) には原稿・ドラフト・一次資料を置く。

## 4. 命名規則

- 日本語文書は `.ja.md` を付ける。
- 英語文書は `.md` を付ける。
- 新規文書は日時付きの一意名を優先し、長期運用文書は短い安定名を使う。
- 既存の文書名を変える場合は、参照先の更新も併せて行う。

## 5. 更新責任

- 仕様が変わったら、関連する README / AGENTS.md / docs 系文書の参照先を更新する。
- ノートの内容が公式文書に移ったら、ノート側では「移行済み」または「参照先更新済み」と明示する。
- 重要な更新は、関連するハブ文書に反映する。

## 6. 状態ラベル

- `[DRAFT]` : まだ整備途上
- `[REVIEW]` : レビュー待ち
- `[CANONICAL]` : 正規化済みの主文書
- `[TRANSFER-QUEUED]` : 転送候補
- `[TRANSFERRED]` : 他リポジトリへ反映済み
