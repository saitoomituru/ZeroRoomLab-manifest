# Docusaurus / プログラム部分アップデート計画

作成日: 2026-07-13

## 1. 目的

quantaril_cloud_Q3 の実装側では、生成コード・Docusaurus 構成・コンテンツ生成フローを分離し、ノート由来の構造変更を追跡しやすくする。

## 2. 対象範囲

- generate_*.py 系の生成ロジック
- Docusaurus の設定・サイドバー・README
- コンテンツ生成に関わる CI / デプロイ手順

## 3. 改修方針

- 生成ロジックは「内容生成」と「文書配置」を分離する。
- docs / blog / about といった出力先を明示的に管理する。
- ハブ文書とストリーム文書の構造を、実装側でも理解しやすいように整理する。

## 4. 優先度

1. 生成スクリプトの責務分離
2. Docusaurus 構成の整理
3. 既存の README / サイドバー参照の更新
