## 思想：主権の回収

800系は、次の思想で統一されている。

撮影・配信・演出・計測は、通常はスタッフ、SaaS、機材メーカー、外部ラボに分散している。
800系はそれを**演者/実験者本人の身体周辺**に取り戻す。

```
OND800            = 撮影空間の主権
FAN800            = 物理演出空間の主権
SAO800            = 配信母艦とSaaSフェールオーバーの主権
DVE800            = 編集空間の主権
PSYCHO-Py800MCP   = 計測と実験ログの主権
```

外部に一言で説明するなら：

> 800系は、演者・実験者がソロでマルチカメラ配信、物理演出、OBS母艦処理、
> 実機計測を制御するための、ローカルファーストな分散コックピットシステムである。

## OND800：撮影コックピット

Raspberry Pi 5を中心にしたモビルスーツ型コックピット。
NDIマルチカメラ、OBS制御、FAN800へのイベント配布を担当。

**目的：** 演者本人に撮影空間の主権を戻す。
演じながら片手でNDIマルチカメラ制御、BLE IoTファンネル展開、マルチRTMP兵装切替を実行できる。

良い点：
- 目的が明確。ビジョンが技術構成に直結している
- レイヤー分離が強い。SaaS、OBS、OND800、FAN800を明確に分けている
- `docs/interface_spec.md` がFAN800・SAO800の実装正本になっており、仕様漂流を防ぐ構造

---

## FAN800：物理演出ファンネル

ESP32ベースの自律演出IoTプラットフォーム。
BLE GATTでOND800からイベント言語を受け取り、照明・特効・MIDI・DMXなどの物理演出を実行する。
OND800が落ちてもFAN800はループ継続し、物理安全チェックはFAN800が最終防衛ラインを守る。

**思想：** 「黒子をESP32に置き換える」。
現場のAD、黒服、照明卓、MIDI、DMXの手動段取りをファームウェア化する。

設計の特徴：
- BLEと内部クロックの責務分離が正しい（BLEはルール配布、リアルタイムタイミングはFAN800内部RTCが取る）
- 物理安全を末端に置いている（上位が暴走しても末端が拒否できる）

---

## SAO800：配信母艦拡張

OBSに寄生する母艦拡張。
OND800が演者用コックピットなら、SAO800は後方で重演算・兵装ビルド・フェールオーバー管制を引き受ける。

担当機能：
- RTSP→NDI変換（GStreamer経由）
- BPM解析
- NSFWサニタイズ（ローカル編集でSFW表現へ変換）
- AIオフロード
- フェールオーバー制御

設計の特徴：
- OND800の認知負荷を増やさない設計（OND800はスロット番号だけ投げ、SAO800が重処理を引き受ける）
- OBSに寄生し、自前レンダラーを作らない原則

---

## DVE800：編集エンジン

DaVinci Resolve（無料版）をローカルAIで強化する、完全ローカル動作の動画編集自動化エンジン。800系の「編集レイヤー」。

**目的：** NSFW・専門ドメイン・日本語特化コンテンツの制作ワークフローを完全ローカルで自動化する。GPUベンダーの言論ポリシー・クラウド審査・規約に依存しない。

担当機能：
- Whisper-Lによる話者別FAM多義性マップ注入型の音声書き起こし
- DaVinci Resolve APIブリッジ（タイムライン自動構築・SRTインポート）
- 審査フィードバック（ビデ倫等）の正規化・ショットマッチング
- モザイク検出候補の提示（強度の最終決定権は常にオペレーター側）

設計の特徴：
- 検出モデルの出力を「最終解」にしない。「担げるロマン砲」思想——超兵器ではなく人が実際に運用できる規模で作る
- FAM実装はFAMoverMCPのトイモデル検証段階（詳細は [dve800.ja.md](dve800.ja.md) 参照）

## PSYCHO-Py800MCP：計測器MCP

オシロスコープ・ロジアナ・WebCam・顕微鏡をMCPで統合し、
AIが「波形の眼」を持てるようにするシステム。

```
人間の担当：プローブを当てる・GNDを取る・最初のStartを押す・破壊方向操作を承認する
AIの担当  ：成立済み観測の読取・保存成果物の解析・未来のRisk／Benefitと不足能力の提示
```

設計の特徴：
- ユーザー成立済みの測定読取は観測支援として扱い、AIによる測定開始・入力経路変更は計画と二者確認へ分離
- 現在・進行中incidentは枯れたwarning GUI、測定前の未来projectionはSF／幻想themeで表示
- 幻想paneは現在の警報・実測・課題を軽くせず、覆わず、消さない
- 安全を責任の押し付けでなく、事故前に人員・装備・時間・停止復旧能力を組む工学として扱う
- 証拠保全と中止後の部分成果物確定を設計へ含める

上位哲学は[ちくわ砲 — 幻想化によるProspective Meta-Safety](../philosophy/chikuwa-cannon-prospective-meta-safety.ja.md)を参照。

## ローカルファースト設計

SaaSは便利な兵装だが、唯一の生命線ではない。
NDI、BLE、USB-MIDI、HDF5、SQLite、ローカルLLM、OBS、GStreamerを使い、
ネットワークやSaaSが死んでも最低限の出撃・収録・測定が継続できる構造。

---

## 比喩が設計仕様になっている

モビルスーツ、ファンネル、母艦、波形の眼といった比喩は飾りではなく、
**責務分離のための認知UI**になっている。

```
OND800       = モビルスーツ型コックピット（演者の身体拡張）
FAN800       = ファンネル（自律飛行する物理演出ユニット）
SAO800       = 母艦（後方で重処理を引き受ける）
DVE800       = 編集エンジン（OLYMPIA G-8000Sの系譜を継ぐローカル編集機）
PSYCHO-Py800 = 波形の眼（AI計測・観測の実装）
```
