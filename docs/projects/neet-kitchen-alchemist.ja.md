# NeeT_kitchen_ALCHEMIST — 人間側の生活兵站研究

状態: `[CANONICAL]`（ワークスペース境界・目的） / `[EXPERIMENTAL]`（個別レシピ）  
制定日: 2026-07-24  
対象ワークスペース: `NeeT_kitchen_ALCHEMIST`  
最初の実装: [OpenSourcePITETO](https://github.com/saitoomituru/OpenSourcePITETO)

## 1. 何を研究するか

`NeeT_kitchen_ALCHEMIST` は、低資源の生活者が家庭設備、公開知識、身体感覚を組み合わせ、食べられる成果物を作るための生活兵站研究ワークスペースである。

ここでいう兵站は、軍事用語の借用だけではない。人間が生存し、研究し、次の一日へ到達するために必要な次の流れを扱う。

```text
入手できる食材
    ↓
家庭にある熱源・器具
    ↓
人間の手・目・鼻・口による制御
    ↓
食べられる成果物
    ↓
失敗条件を含む公開知識
```

雇用、資本、大規模設備が先に揃うことを実験開始条件にしない。成果物を食べ、再利用可能な知識を社会へ返せたかを、現段階の中心的な目的関数とする。

## 2. 「ニート用バズレシピ」の意味

この棚でいうバズレシピは、広告KPIとしての拡散数を最上位目標にするものではない。

- 安価または入手しやすい材料から始められる
- 一目で試したくなる成果物がある
- 工程を単なる手順ではなく、物理的・食品工学的な役割へ分解する
- 固定値で吸収できない個体差を、人間のフィードバック制御へ返す
- 成功だけでなく、フニャる、焦げる、味が定着しない等の失敗条件を公開する
- 読者が自分の家庭設備へ移植し、改変結果を再び共有できる

という、入口の強さと再現知識を両立する公開レシピ形式を指す。

## 3. Sphere／Deb800との境界

`NeeT_kitchen_ALCHEMIST` は、次のいずれにも属さない独立ワークスペースである。

- **スフィア**: SphereOS、FAM、AQC、IBD等を扱う抽象メタエンジン層
- **Deb800**: 撮影、演出、配信、編集、計測を束ねるx800ベアメタル／現場実装層

思想、比喩、実験作法を参照することはありうるが、ワークスペースの同居や実装依存を意味しない。ポテトチップスを作るためにSphereOS、FAM、OND800、DVE800等を導入する必要はない。

この分離により、AIや高価な装置を生活研究の前提にせず、人間の身体と家庭設備だけで閉じる実験を第一級の研究として保持する。

## 4. ワークスペース構成

`NeeT_kitchen_ALCHEMIST.code-workspace` の現行メンバーは次の2つである。

| メンバー | 責務 |
|---|---|
| `NeeT_kitchen_ALCHEMIST/OpenSourcePITETO` | レシピ、実機写真、画像圧縮ツール、ライセンス |
| `ZeroRoomLab-manifest/ZeroRoomLab-manifest` | 共通の認知規約、ワークスペース境界、公開主張、プロジェクト位置づけ |

同一ワークスペースにあることは、相互のruntime依存、submodule依存、package依存を意味しない。

## 5. 最初の実装: OpenSourcePITETO

`OpenSourcePITETO` は、家庭用設備へポテトチップス製造工程の機能を再配置する実験である。

```text
Fry
├─ デンプンの糊化
├─ 蒸気発生
└─ 多孔質構造の形成

Flavoring
└─ 表面油と残留水分を使った粉末の定着

Oven
├─ 残留水分の追い乾燥
└─ 多孔質構造の乾燥固定

Human-in-the-loop
└─ 5分ごとの試食による終了判定
```

2026-07-24時点のリポジトリ表示は `v0.2 Experimental / EATABLE` であり、家庭用フライヤー／揚げ物鍋、家庭用オーブン設定210℃、5分間隔の試食でポテトチップスになった実機試験を記録している。

210℃はオーブン設定温度であり、揚げ油を210℃へ加熱する指示ではない。個別試験の成功は、すべての品種、スライス厚、油、オーブンに共通する固定時間を証明しない。

## 6. 人間側兵站として記録する軸

今後のレシピでは、必要に応じて次を記録する。

| 軸 | 記録候補 |
|---|---|
| 調達 | もらい物、余剰食材、一般小売、保存状態 |
| 設備 | 鍋、フライヤー、オーブン、包丁、スライサー |
| エネルギー | 熱源、設定温度、加熱時間、余熱、複数工程への分担 |
| 身体 | 味覚、食感、匂い、火傷回避、認知負荷 |
| 品質 | サクサク感、味の定着、焦げ、湿気、回復可能性 |
| 再利用 | 失敗条件、改変点、写真、再現範囲、ライセンス |

すべてのレシピへ同じ計測器を強制しない。主張したい差分より細かい測定が必要な場合に、重量、温度、時間、反復等を追加する。食べられたという実機観測を、計測器不足だけで未実施へ戻さない。

## 7. 現在の境界

### [FACT]

- 公開実装として確認できるレシピは、現時点では `OpenSourcePITETO` のポテトチップス実験である。
- 日本語READMEを正本とし、en-US版をテキサス周辺のgarage lab／DIY文化へ接続する外部GUIとして持つ。
- 公開用JPEGは縮小再エンコードし、Exif/GPS撮影メタデータを除去する。

### [UNKNOWN]

- 次の食材とレシピ
- 複数レシピを一つのリポジトリへ置くか、別リポジトリへ分けるか
- 食費、保存日数、熱源費、栄養等をどこまで共通兵站指標にするか
- GitHub Projects上の管理単位

未作成のレシピをロードマップ上の約束や実装済み機能へ変換しない。

## 8. 参加と改変

参加者は、工場設備の縮小コピーだけでなく、工程の目的を読み、自分の家庭にある別の器具へ移植できる。改変時は、少なくとも次を残す。

- 元にしたレシピとライセンス
- 変えた食材、厚さ、温度、時間、器具
- 成功、失敗、途中状態
- 人間が終了判定した条件
- 写真に残る位置情報等の公開不要メタデータを除去したか

商用利用や再配布は各リポジトリのライセンスに従う。外部支援や商用化が生じても、公開知識、改変可能性、家庭設備への移植可能性を奪うことを既定条件にしない。
