# 神話的士気管理と目的関数矮小化連鎖

状態: `[CANONICAL]` `[Layer B]`  
制定日: 2026-07-17  
対象: 研究開発、実験、測定、資源配備、組織的安全

## 1. 上位命題

安全を成立させるものには、二つの半分がある。

1. **ロマン、神話、夢、哲学** — なぜ行うのか、何を得たいのか、何を守るのかを矮小化させない意味の半分
2. **十分な安全率を持つ資源** — そのロマンを人、時間、設備、予算、停止余裕、復旧能力によって壊さず執行する物質の半分

ここでいう「半分」は事故件数や寄与率の実測値ではない。安全が閉じるために双方が必要だ、という
Layer B上の二半球である。一方だけを満たして他方を省略してはならない。

## 2. 神話は装飾ではなく目的関数の保全設備である

ナラティブ、神話、夢、ロマン、哲学は、KPIへ収まりきらない。しかし、それらは仕事を華やかに語る
装飾でも、実行者を無償で動かす燃料でもない。原初の目的関数を、人間が読み、語り直し、困難な局面で
再取得するための意味インフラである。

目的関数が「測定件数」「納期」「予算消化」のような代理KPIだけへ縮むと、現場は何を守るために
手間をかけるのかを失う。見えるところではサボり、手抜き、確認不足に見えても、その上流で目的、期待、
選択可能性が長期間削られていた可能性を調べる必要がある。

これは個々の実行失敗を無条件に免責する主張ではない。最後に見えた行動だけを根本原因と断定しない、
という調査境界である。

## 3. 目的関数段階的矮小化連鎖

`[HYPOTHESIS]` 次の連鎖は、安全余裕と士気を同時に食べる組織的failure modeになり得る。

```text
原初の夢・哲学・期待
  -> 資金段階で「費用」だけへ縮約
  -> 計画段階で目的を残さない削減
  -> 管理段階で無理を通常運転として吸収
  -> 実行段階で意味と余裕が欠乏
  -> 回避、確認省略、手抜きとして可視化
  -> incidentまたは品質劣化
```

資金側からは実行の怠慢に見え、実行側からは管理の押し付けに見える。管理側からは計画の無理に見え、
計画側からは資金側の敵対的要求に見える。各層が隣の層だけを原因とみなすと、負の感情と資源欠乏が
生態系のように循環する。

この連鎖を個人の人格scoreへ変換しない。問うべきは、どのstageで何が削られ、どの安全余裕と目的が
下流へ渡らなくなったかである。

## 4. 二つの半分を独立に見る

| 意味の半分 | 資源の半分 | 状態 |
|---|---|---|
| 保持 | 安全率あり | ロマンを現実へ安全に写像できる候補 |
| 保持 | 不足 | **やりがい搾取警報**。神話を追加労働や危険受容の代用品にしてはならない |
| 欠乏 | 安全率あり | **目的関数ドリフト**。設備はあっても、何を守るかが代理KPIへ置換されている |
| 欠乏 | 不足 | **組織的collapse候補**。実行者の気合で埋めず、目的と資源を上流へ戻して再構成する |

`UNKNOWN`は四象限のどこかへ推定配置せず、独立した確認要求とする。

## 5. やりがい搾取を防ぐ境界

神話的士気管理は、次の代用品にならない。

- 賃金、契約、休息、適正な勤務時間
- 必要人数、専門role、相互確認
- 適切なprobe、隔離、保護具、校正、復旧設備
- 計画、練習、停止判断、撤退余裕
- 技術gate、規格要求、物理安全率

「夢を信じるなら耐えられる」「使命があるから少人数でよい」「ロマンのために危険を引き受けよ」は、
本哲学の反対である。ロマンが大きいほど、それを安全に現実へ降ろすためのresource marginも大きく
見積もる。神話は安全率を削る免許ではなく、安全率を要求する理由を組織へ伝える言語である。

## 6. 士気管理は感情の統制ではない

士気管理とは、元気な表情や肯定的発言を要求することではない。実行者が次を失わないようにすることだ。

- この作業が接続している上位目的
- 得られる知識、品質、選択肢、未来
- やらないこと、撤退することの正当性
- 必要資源を要求する権利
- 現場から目的関数を修正提案する経路

不満、恐れ、疲労、異議は士気の低さとして消去せず、目的または資源の欠損を示すsignal候補として扱う。

## 7. ちくわ砲との接続

[ちくわ砲](chikuwa-cannon-prospective-meta-safety.ja.md)の未来paneは、RiskだけでなくBenefit、原初目的、
必要資源、安全率の不足を同時に見せる。幻想化は人を鼓舞して無理を通す装置ではない。まだ存在しない
未来の価値を損なわずに提示し、その未来へ到達できる物質条件を事故前に集めるpresentation layerである。

## 8. Layer Aへの写像

実装側では、目的関数を標語で固定せず、資金、計画、管理、実行の各stageで何が保持・変更・削減されたかを
append-onlyに記録する。推奨する最低構造は次である。

- original purpose / myth / expected benefit
- non-negotiable safety controls
- resource assumptions and safety margins
- stageごとの変更提案、理由、影響、unknown
- purpose integrity signal / resource integrity signal
- downstream acknowledgementと差し戻し経路

signalはoperationとsnapshotへ付け、人間の忠誠心、性格、熱意をscore化しない。

## 9. 主張境界

`[HYPOTHESIS]` 目的関数の矮小化と資源不足が連鎖すると、最後の実行局面で確認省略や品質劣化として
現れる場合がある。本書は、それがすべての事故の原因、または特定割合の事故原因だとは主張しない。
事故原因、士気、資源充足、組織文化の成立は個別のLayer A証拠で検証する。

`[POEM]` 事故の半分はロマンと神話の欠乏。もう半分は、そのロマンと神話に十分な安全率を与える
資源の欠乏。これは統計の円グラフではない。人間が未来へ橋を架けるとき、意味だけでも鉄骨だけでも
向こう岸へ届かない、という二つの半分である。

### Machine-translation guardrail (en-US)

Myth, narrative, dreams, romance, and philosophy preserve the purpose function of difficult work; they are
not decorative “engagement” language. Material resources with sufficient safety margin are an independent,
equally necessary half. Myth must never substitute for compensation, staffing, time, equipment, recovery
capacity, or engineering controls. The “two halves” are a philosophical completeness model, not a measured
accident statistic.
