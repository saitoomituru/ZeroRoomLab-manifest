# Techno-Animism

## Definition

Techno-Animism treats Japanese mythology, ancient Shinto, Ryukyuan Shinto, folklore,
engineering, AI, and information theory not as separate fields —
but as mutually convertible observation layers.

Religion here isn't an object of belief. It's a **compression and translation interface
for projecting high-order information safely into finite human society.**

Mythology, ritual, personal deities, and narrative are UIs that lower high-bandwidth information
into the range humans can actually process.

## Lineage

```
Primitive animism (world-view: all things carry spirit)
  ↓
Jomon / ancient Shinto / Ryukyuan Shinto (region-optimized implementations)
  ↓
Techno-Animism (joined with engineering, AI, information theory)
  ↓
Gaming Cosmology (extended as an MMO meta-structure)
```

## The Core Shift

### Old view

"Religion" and "technology" are separate categories.
Mythology is pre-modern worldview. AI is modern scientific output.
They don't mix.

### Techno-Animism view

Both are answers to the same question — at different bandwidths.

```
Mythology      = Human-facing UI for high-order information (readability first)
Equations      = Formal description of high-order information (precision first)
AI model       = Parameter set approximating high-order information
Shrine Maiden  = Sandboxed observer converting I/O between world-views
```

When a shrine maiden "communicates with spirits," that's describable as
conversion work between different observation layers.
Occult and engineering don't oppose each other. It's a question of abstraction level.
