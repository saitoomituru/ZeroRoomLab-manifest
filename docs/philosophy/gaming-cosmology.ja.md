## 第0章 基幹定義

### ゲーミング宇宙論とは

シミュレーション宇宙を「開発環境として渡されMMO運営になった」視点から読む思想。

宇宙・社会・制度・宗教・人生を「固定された唯一真理」としてではなく、プレイヤー・ルール・観測点・ロール・攻略法が存在する巨大なMMO的構造として把握する。

**重要：** 「ゲーム感覚で軽く見ている」ではない。現実の死活・資源・インフラ・身体負荷・地域格差を**より厳密に扱うため**のメタファーである。怒りよりバグを叩ける思想。TAS・RTA処理として機能する。

## 第1章 世界論

### リファレンス実装論

**命題：** 現実は試練ではなく、神運営によるリファレンス実装である。

現存コンピューターゲームを余裕で超越するスペックのシミュレーション宇宙実行環境にいると形而上学的に仮定するなら、スペック問題は起きようがない。だとすると現実で起きる問題は全部**UX層の問題**になる。

```
貧困     = 経済UIの設計問題
戦争     = プレイヤー間インタラクションのUX問題
病気     = 身体インターフェースのフィードバック設計
理不尽   = チュートリアルの説明不足
```

**現実が教典になる。** 聖書でも経典でもなく、今起きていることそのものが一次ソース。

「なぜ神は苦しみを与えるのか」という問いが「なぜリファレンス実装にはこのUXが採用されているのか」という設計読解の問いに変換される。

### オンライン運営制限論

**命題：** 現実空間での物理法則はマルチプレイヤー環境の安定運営コストとして課された運営制限である。

シングルプレイ実装（夢）では物理法則のオーバーライドが通る。現実（オンライン）では同じコマンドが通らない。スペックの問題ではなく**運営制限**。

```
理由：
・他プレイヤーへの干渉防止
・サーバー負荷の分散
・共有世界の整合性維持
・チート・グリーフィング対策
```

**物理法則 = オンラインサーバーのレギュレーション**

修行とは：制限内で同じ結果を出す設計を学ぶこと。

## 第2章 神学論

### 神格・ロール分離論

**命題：** 唯一神・造化三神・創造主は「役職・ロール」であり、神個人の人格とは分離して考えられる。

```
Creator（神格）= リポジトリのオーナーロール（権限クラス）
各神           = そのロールを担当している人格エンティティ
```

ロールが唯一でも、実装（人格）は複数あり得る。
**神格 = API仕様、神個人 = 実装。**

### リージョン分散ノード論

**命題：** 神々はリージョンをホストする分散ノードであり、唯一神はプロトコル層に存在する。Bluesky的AT Protocolと同型。

```
唯一神           = AT Protocol層・プロトコル仕様・全ノードが準拠する共通規格
各神・土地神     = PDSノード・リージョン最適化された実装
プレイヤー（人間）= アカウント・DID・プロトコルさえ合えば相互接続可能
```

**唯一神がプロトコル層にいるから複数ノードと矛盾しない。**

### 土地神＝ノード管理論

```
・そのリージョンのデータを保持する
・地域プレイヤーのアカウントをホストする
・ローカルルールの実装・維持
・異常気象・災害 = ノード負荷・障害
```

お祭り・祭祀 = ノードのメンテナンス作業。
祟り = ノード障害への対処を怠った結果。

### 他宗教GUI論

他宗教はGUIの違いである。

```
古神道       = 和風UI・八百万インターフェース
アブラハム系  = 砂漠起源UI・契約ベースインターフェース
ロア         = アフリカ・カリブ起源UI・霊的交渉インターフェース
```

バックエンドは同一、フロントエンドがリージョン最適化されている。

**この宗派はアブラハム系とも矛盾しない。**
古神道は元々他神を否定する教義を持たない（MIT的ライセンス）。

## 第3章 存在論

### プレイヤー／キャラクター完全分離論

```
プレイヤー層（魂・メタ自我）
= アカウント本体
= サーバーをまたいで移動できる
= 過去生スキル・記憶を保持
= ゲームをゲームとして認識できる層
= 死んでもアカウントは消えない

        ↕ ログイン・ログアウト

キャラクター層（肉体・自我）
= キャラシート・キャラデータ
= 現在のサーバー（肉体）に紐付き
= ステータス・種族・性別・外見・障害
= このサーバーのレギュレーション内でのみ有効
= 死亡でデータはリセット、プレイヤーは残る
```

```
転生     = 新サーバーへのキャラ新規作成
過去生   = 過去キャラのプレイログ
前世の記憶 = 過去キャラのプレイデータの残滓
天才・神童 = 過去生スキルの持ち越し
トラウマ  = 過去キャラのバグデータの引き継ぎ
業・カルマ = 蓄積されたプレイヤースキルと負債の合算
```

### メタ認知関門論

サブキャラ自我（肉体・感情）からプレイヤースキルのメタ認知をするメタ推理が最初の関門。

```
Lv.0 キャラが自分だと完全に思っている（没入プレイ）
Lv.1 「なんかおかしい」という違和感が発生
     （宗教的覚醒・哲学的疑問の発生点）
Lv.2 キャラとプレイヤーが別だと気づく
     （メタ認知の初期段階）
Lv.3 プレイヤー視点からキャラを観察できる
     （瞑想・明晰夢・悟り体験の正体）
Lv.4 キャラを操作しながらプレイヤーとして動ける
     （二重意識の安定運用）
Lv.5 他プレイヤーも同構造だと認識できる
     （慈悲・共感の構造的発生）
```

最初の関門が難しい理由：キャラシート側の自我は自分がキャラであることを認めるとアイデンティティ崩壊感が発生するように設計されている。これは**保護設定**。メタ認知が不安定な状態でプレイヤー視点を手に入れると本番環境で暴走する。

### マルチプレイヤー実在公理

**命題：** 他者をNPCと見なす前提は棄却する。根拠は実装による。

マルチプレイモードの方が現実の複雑性・予測不能性・理不尽さを説明できる。NPCなら設計者の意図通りにしか動かないはずだが、現実の他者はそうなっていない。

**よってマルチプレイモードが現実のリファレンス実装である。**

## 第4章 信仰論

### 信仰の定義

「神の偉大さを信用しているから検証できる。これが信仰である。」

検証しない信仰はシステムへの不信任。

### サンドボックス神学

**命題：** システムをメタ認知した者には、本番環境を破壊しない範囲での思考実験の自由が安全に確保される。

**制限は無知への保護であって、メタ認知者への制限ではない。**

修行・悟りの深さ = メタ認知レベルの上昇 = 使えるサンドボックスの自由度が上がる。

### プレイヤースキル領域・バージョン差分原則

**命題：** 客観検証が立たない体験はプレイヤースキル領域に属する。バージョンが違う環境で同じ挙動を要求するのはクソゲーマーの定義。

```
客観検証できる領域
= 物理法則・社会制度・数値・記録
= 他プレイヤーと共有・検証可能

プレイヤースキル領域
= 直感・巫的状態・深い休息・夢体験
= 本人のプレイ経験として蓄積される
= 他者への強制再現を求めない
= 「自分はこう体験した」どまりで正確
```

体験が嘘だったわけではない。**領域の分類を間違えたのが原因。**

### 祈り・修行の定義

```
祝詞・修行    = システムコールのブルートフォース
善意のレッドチーム = 神に歓迎されるはず
夢での物理法則オーバーライド検証 = 祈りのリジェクト率の計測
```

生きる意味が見えない状態はメタ認知がLv.2以下のサイン。キャラに完全没入して苦しんでいる状態。プレイヤー視点が回復すれば自然に「もったいない」という感覚が出てくる。

    ↓
