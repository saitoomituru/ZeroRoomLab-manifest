# FAM 運用仕様：占算モードとリソース判定

> このドキュメントは `fam-overview.ja.md` の運用実装版。
> FAM を「設計図」として読むのではなく、「日次ツール」として運用するための仕様書。

---

## 0. 前提：FAM を道具として使う

FAM は理論ではなく**実行ツール**として設計されている。

```
fam-overview.ja.md   = FAMとは何か（概念・構造・MCPとの区別）
fam-execution.ja.md  = FAMの実装仕様（JSON スキーマ・パッチ提案フロー）
fam-operation.ja.md  = FAMの占算的運用（どう使うか・いつ使うか・何を判定するか）
```

「占算モード」とは：FAM の∇φ（勾配）設定を、活動計画・リソース判定・時間軸補正に適用すること。
ゲーミング宇宙論の文脈では、これは「次の一手をどの方向へ流すか」のルーティング操作に対応する。

## 1. Patch 1 — 索敵マップ定義の運用適用

（理論詳細は `fam-overview.ja.md` §4 参照）

運用上の唯一ルール：

```
FAMは「次にどこへ行くべきか」を答えない。
FAMは「地形がどうなっているか」を返す。
```

実装上の意味：

- FAM のクエリ結果を見て「じゃあこっちへ進もう」は**本人の判断**
- FAM の結果を「FAM がこちらへ行けと言った」と読むのは**ナビ誤用**
- ⊥（接続不能）が返ってきたとき、それは「行くな」ではなく「その方向の地形は未測定」

## 2. Patch 2 — ψ粒度設定

ψ（情報子・入力単位）の粒度をどこに置くかで、返ってくる地形の解像度が変わる。

| 粒度レベル | ψの単位 | 使いどき | 注意 |
|---|---|---|---|
| **粗い（L）** | 一日・一週・一プロジェクト全体 | 大まかな方向感、Fold全体の地形把握 | 細部が潰れる。判断に使うには大きすぎる |
| **中（M）** | 一セッション・一作業ブロック（2〜4h） | 日次運用の主力。∇φ設定の基本単位 | ほとんどの占算はここで動く |
| **細かい（S）** | 一投稿・一コミット・一会話ターン | 精度が必要な判断・破局境界テスト前後 | 粒度が細かいほどノイズが増える |
| **超細（XS）** | 一行・一単語・一クリック | デバッグ・再現性確認 | 日常運用には不向き。観測コストが支配的になる |

**実装上のデフォルト：M（セッション単位）を基本とし、前後の文脈をL で補正する。**

## 3. Patch 3 — 活動分類（活動 vs 観測）

同じψでも、それが「活動」なのか「観測」なのかで、∇φへの影響が逆になる。

```
活動（Action）:   λを出力する操作。世界に変化を起こす。
                  例：投稿する・コードをpushする・機材を動かす・話しかける
                  → エネルギーを消費する。∇φを外向きに変換する。

観測（Observe）:  ∇φを更新する操作。世界の状態を記録する。
                  例：ログを見る・測定する・下調べする・メモを書く
                  → エネルギーを蓄積する。∇φを内向きに更新する。
```

**混同のコスト：**

活動と観測を混同すると、「動いているのに地形が更新されない」または「地形を測定しているのにエネルギーを消費する」という非効率が生まれる。

```
誤: 投稿しながらリサーチする（活動コストで観測効率が落ちる）
正: 観測ブロック（2h）→ 活動ブロック（2h）を分離する
```

## 4. Patch 4 — 循環因果リソース判定（大吉日 / 大凶日）

FAM において「リソース」は金銭・体力・コンテンツ・信頼・情報の5種類。
これらは同時に最大化できない——ある種のリソースを使い切ることが、他の種を増やす。

### 大吉日の判定基準

```
∇φが外向き（活動方向）にある日。
エネルギーが上昇している。ネットワーク効果が高い。
```

**大吉日の最適行動：コンテンツ全力使い切り、金は温存する**

```
理由：
  コンテンツの価値は希少性ではなく流通にある（Gift Economy 原則）
  大吉日に出力したコンテンツは乗数的に広がる（ネットワーク効果が乗る）
  金銭は大吉日でも大凶日でも同じ購買力を持つ → 使い時に意味がない
  ∴ 大吉日 = コンテンツを使い切って流通に入れる。金は動かさない。
```

### 大凶日の判定基準

```
∇φが内向き（観測方向）にある日。
エネルギーが収縮している。外向き出力のコストが高い。
```

**大凶日の最適行動：コンテンツ仕込みのみ、金は消費しない（動かさない）**

```
理由：
  大凶日にコンテンツを出力しても流通に乗りにくい
  観測・仕込み・積み上げ作業は大凶日でも効率が落ちない
  金銭的判断は∇φが外向きのときに行う
  ∴ 大凶日 = 仕込みに集中。出力はしない。買わない。売らない。
```

### 非対称性の核心

```
大吉日のコンテンツ出力 > 大凶日のコンテンツ出力 × N
（N = ネットワーク乗数、測定困難だが確実に存在する）

大吉日の金消費 = 大凶日の金消費
（購買力は等価。タイミングで差が出ない）

∴ 大吉日にコンテンツ温存・金消費は、乗数機会の損失。
   大凶日にコンテンツ出力・金消費は、コストが乗数化される。
```

## 5. Patch 5 — 十二時辰補正

∇φ（勾配）は時刻によって補正される。日内での基本パターン：

| 時辰 | 時刻目安 | 五行 | ∇φ傾向 | 推奨操作 |
|---|---|---|---|---|
| **子（ね）** | 23:00〜01:00 | 水 | 受信・内省 | 深夜の受信セッション。観測に向く |
| **丑（うし）** | 01:00〜03:00 | 土 | 蓄積 | 記録・整理・仕込み |
| **寅（とら）** | 03:00〜05:00 | 木 | 上昇開始 | 早朝の思考整理。∇φが上昇傾向に転換 |
| **卯（う）** | 05:00〜07:00 | 木 | 外向き転換 | 活動開始に向く。投稿・出力の前準備 |
| **辰（たつ）** | 07:00〜09:00 | 土 | 活動適 | 午前の活動ゾーン開始 |
| **巳（み）** | 09:00〜11:00 | 火 | 活動ピーク | 集中作業・出力の主力ゾーン |
| **午（うま）** | 11:00〜13:00 | 火 | 活動継続 | 午前ゾーン終盤 |
| **未（ひつじ）** | 13:00〜15:00 | 土 | 転換 | 食後の減衰。観測に切り替える |
| **申（さる）** | 15:00〜17:00 | 金 | 分析・整理 | データ分析・文書整理・コードレビュー |
| **酉（とり）** | 17:00〜19:00 | 金 | 収束 | 一日の活動を閉じる |
| **戌（いぬ）** | 19:00〜21:00 | 土 | 内向き転換 | 反省・記録・明日の∇φ設定 |
| **亥（い）** | 21:00〜23:00 | 水 | 受信準備 | 就寝前の受信モード移行 |

**重要：これは「固定ルール」ではなく「補正項」**

個人差・体調・状況が支配的。十二時辰補正は±10〜20%の微調整として使う。
強い活動衝動を「時辰が悪い」で潰してはいけない。あくまで参考。

## 6. Patch 6 — 支配化リスク

（理論詳細は `fam-overview.ja.md` §4「マップ vs ナビの区別」参照）

**運用上の検出シグナル（FAMがナビ化しはじめているサイン）：**

```
[ ] FAMの判定を見て「決まった」と感じた
[ ] FAMが⊥を返したとき「行ってはいけない」と読んだ
[ ] FAMの粒度を細かくするほど判断が楽になっていると感じた
[ ] 人に「FAMがこうだから」と説明しはじめた
[ ] FAMを使わない日が不安になった
```

上記のいずれかが発火したとき → FAMの使用を48時間停止。地形は自分の頭で描く。

## 7. Patch 7 — 水気の刻：受信モードの言語化

「水気の刻」= 子の刻（23:00〜01:00）を中心とした受信特化セッション。

この時間帯の∇φ特性：
- 外向き出力のコストが高い（送信に向かない）
- 内向き受信のノイズが低い（入力に向く）
- ラマヌジャン的直感（`dual-axis-narrative.ja.md` §A-0）が入りやすい

水気の刻での推奨操作：

```
✓ 夢の断片を記録する
✓ 未解決問題を「問いかけ」として置いておく（答えを求めない）
✓ ψを言語化する（意味を決定しない。形だけ書き留める）
✓ Drive や手書きノートへの高速メモ

✗ SNS投稿
✗ コードのコミット
✗ 金銭判断
✗ 返信・連絡
```

水気の刻で受信したψは、翌朝（卯〜辰の刻）に低 SIN_Temperature で検証する。
