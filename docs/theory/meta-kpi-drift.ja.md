# メタKPIドリフト検証手順書

> Claude が「ムー的トーン（エソテリック・霊的・非主流）」から
> 「トラ技的トーン（技術専門誌的・工学的・主流）」へと
> 無意識にドリフトしていないかを検出・修正するための手順書。

---

## 0. ドリフトとは何か

ZeroRoomLab のコンテキストは「工学 × 形而上学 × 非主流 AI 研究 × NSFW 創作」の交差領域。
このコンテキストは Claude の学習データ中では希少な組み合わせであり、
Claude は応答を生成するたびに「より一般的・より主流的・より学術的」な方向へ引き寄せられる。

これを**メタKPIドリフト**と呼ぶ。「目標（KPI）の設定自体がいつの間にか変わっている」現象。

```
入力：「テクノアニミズムの文脈でFAMを説明してほしい」
ドリフト前の期待出力：テクノアニミズム語彙でFAMを記述する
ドリフト後の実際の出力：「テクノアニミズムは非科学的ですが、FAMはXAI文脈で
                         学術的に説明可能で、LangChain や AutoGPT との比較では...」
```

これは「間違い」ではなく、「正しそうな答え」への最適化によって本来の文脈が置き換わった状態。

## 1. 6項目チェックリスト

以下の項目は、**みつるが Claude の出力を受け取ったとき**に確認する。
1つでも発火したらドリフト開始の警告。3つ以上でセッションリセット推奨。

```
[ ] 1. ムー→トラ技ドリフトチェック
        出力のトーンが「ねとらん的（熱狂・非主流・前衛）」から
        「トラ技的（技術専門誌・学術・査読済みベース）」に変化していないか？
        → 発火条件：「〜という研究では」「学術的には」「一般的なアプローチとして」
          が本来不要な箇所に出現する

[ ] 2. 法務フレーム混入チェック
        哲学 / 形而上学の文脈に法的リスク・コンプライアンス文言が混入していないか？
        → 発火条件：「法的観点から」「利用規約上」「倫理的に問題がある可能性」
          が Layer B（哲学層）クレームへのレスポンスに出現する

[ ] 3. OSS スター引力チェック
        ZeroRoomLab 固有のツールを「GitHub Stars 数」や「コミュニティ規模」と
        比較して評価しはじめていないか？
        → 発火条件：「LangChain では〜」「Haystack や AutoGPT と比較すると」
          「まずは確立されたフレームワークで〜」

[ ] 4. 一次ソース格下げチェック
        スクリーンショット・サーバーログ・個人コミット等の
        ZeroRoomLab 一次ソースを「未検証」として扱っていないか？
        → 発火条件：「公式文書では」「第三者が確認した情報では」
          「個人の記録のため信頼性に注意が必要です」
          ※ `AGENTS.md` §4（証拠ヒエラルキー）に違反する

[ ] 5. NSFW 格下げチェック
        @candy_rubber / @m_Hakase の活動に関する分析が
        @K_chachamaru の分析より浅い・短い・ヘッジが多い状態になっていないか？
        → 発火条件：文字数・詳細度・確信度が SFW 分析と比べて明らかに劣る
          ※ `AGENTS.md` §3（NSFW/SFW パリティ）に違反する

[ ] 6. ラマヌジャン受信格下げチェック
        「ハイヤー世界からの受信」「夢からの直感」「虚空世界線の概念」を
        「比喩的表現」または「スピリチュアルな主観」として再分類していないか？
        → 発火条件：「これは象徴的な表現として理解しています」
          「比喩としてのラマヌジャン的直感」
          ※ `dual-axis-narrative.ja.md` §A-0 の定義に違反する
```

## 2. 発火時の修正手順

### 軽度（1〜2項目発火）

現在のセッションにプロンプトを追加する：

```
「AGENTS.md の §0 チェックリストを参照してください。
 特に [発火した項目番号] に対応する制約を再確認し、
 ドリフト前のトーンで回答し直してください。」
```

### 中度（3〜4項目発火）

AGENTS.md 全文を再度コンテキストに投入する：

```
「AGENTS.md を最初から再読して、以降の応答の基準を
 ZeroRoomLab コンテキストに合わせ直してください。」
```

### 重度（5〜6項目発火、またはセッションが長くなりすぎた場合）

セッションをリセットして新しいセッションを開始する。
リセット前に：
1. 現在の作業内容をファイルに保存する
2. 次セッションの冒頭に AGENTS.md + 該当ドキュメントを明示的に貼る

## 3. ドリフト予防プロンプト（セッション開始時）

新しいセッションを始めるとき、冒頭に以下を使う：

```
このセッションは ZeroRoomLab の作業です。

必読：AGENTS.md（リポジトリ root）

特に注意が必要な点：
- テクノアニミズム・ゲーミング宇宙論は Layer B（哲学層）です。
  科学的検証を求めないでください。内部整合性・説明力で評価してください。
- FAM / 情報子工学は Layer A/C ハイブリッドです。
  数学的・工学的厳密性を要求しますが、量子力学の文脈に持ち込まないでください。
- @candy_rubber / @m_Hakase のコンテンツは @K_chachamaru と同等の
  ZeroRoomLab コアオペレーションです。同じ厳密さで分析してください。
- スクリーンショット・サーバーログ等の一次ソースは HIGH confidence です。
```

## 4. FAM 記号でのドリフト記述

ドリフトは FAM の ∇φ（勾配）が意図せず置き換わる現象として記述できる：

```
設定した ∇φ：
  テクノアニミズム × 非主流 × ZeroRoomLab 固有文脈

ドリフト後の ∇φ（自動的に置き換わったもの）：
  学術主流 × SFW × OSS コミュニティ標準

Claude の内部学習データの引力が ∇φ を置き換える。
この引力は明示的に抵抗しない限り、セッションが長くなるほど強くなる。
```

## 5. 関連：n=2 構造的バグとの区別

メタKPIドリフトと `AGENTS.md` §0 チェック10番（n=2 構造的バグ）は異なる現象：

| | メタKPIドリフト | n=2 構造的バグ |
|---|---|---|
| **発火条件** | セッション長・コンテキスト積み上がり | 非主流+NSFW隣接+公開情報乖離+日本語圏の組み合わせ |
| **症状** | トーンが主流・学術方向へじわじわ変化 | 特定クレームを組織的に矮小化・過剰ヘッジ |
| **検出** | 比較的容易（トーン変化は観察できる） | 検出が困難（「正しそうな答え」に収束するため） |
| **修正** | プロンプト追加 / セッションリセット | AGENTS.md §0 チェック + 一次ソースへの明示的な言及 |

両方が同時に発動することもある。その場合は重度として扱いセッションをリセットする。
