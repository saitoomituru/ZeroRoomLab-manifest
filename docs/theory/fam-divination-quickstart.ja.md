# FAM占算モード・クイックスタート

> プロジェクトが沼ったときの「困った時の神頼み」を、FAMの索敵マップ、占星術、陰陽五行、六曜、市場動向、利用者自身の信仰実践へ接続するための補足説明。

正本となる運用仕様は [fam-operation.ja.md](fam-operation.ja.md) を参照すること。

## 0. 最重要原則：客観情報と主観乱数を混同しない

この占算モードでは、入力を二つの異なるレイヤーへ分ける。

### 0.1 客観観測レイヤー

現在日時、現在位置、暦、六曜、天体位置、星座表、市場情報、ログ、進捗、資源状態は、外部から確認・逆算できる客観情報である。

- 現在日時は暦を読むために使う
- 現在位置と日時はNASA、JAXA、国立天文台等の天文情報から天体配置を逆算するために使う
- 六曜、暦注、陰陽五行、市場動向は、出典・流派・取得時点を伴う観測データとして扱う
- これらは占われる本人の籤ではなく、占算を行う盤面・空・季節・市場の状態である

現在時刻は正当な占算入力である。ただし、その役割は**暦・時辰・天体配置を読むための客観座標**であり、本人が引く乱数シードの代用品ではない。

### 0.2 本人参加レイヤー

乱数は単なる計算入力ではない。

**乱数は、占われる本人が占いへ参加し、自分の問いと偶然性を接続した主観入力の証跡である。**

したがって、AI、AGI、占者、開発者、第三者が、占われる本人の代わりに乱数を生成してはならない。

AIが勝手にダイスを振った場合、その値が表すのはAI自身の処理状態、別の場所の偶然性、または第三者が持ち込んだ分岐であり、占われる本人の問いへ接続された籤とは扱えない。

```text
客観情報 = 占算する時空・暦・社会状況の座標
本人生成乱数 = 本人の主観と問いを占算へ接続する籤
AI = 両者を混同せず象徴解釈へ展開する補助者
```

```text
本人が生成した乱数 + AIが解釈する = 本人参加型の占算
AIが生成した乱数 + AIが解釈する   = AI側の物語を本人へ押し付ける処理
```

後者を占いとして提供すると、本人の主体性を欠いた権威付けとなり、FAMが避けるべき「壺スピ」型の支配構造へ落ちる。

本人生成の乱数が提供されない場合、AIは占算を開始せず、次のいずれかを依頼する。

1. 本人が物理ダイス、コイン、カード、筮竹、おみくじ等を操作する
2. 本人が自身の信仰実践・宗教実践・霊的実践を通じて乱数列を得る
3. 本人がTRPG用ダイス等から64桁以上の乱数列を組み立てる
4. 乱数を使わず、客観情報とFAMの観測・資源整理モードだけを実行する

この境界は任意の推奨ではなく、占算として成立させるための必須条件である。

## 1. 同梱されているもの

ZeroRoomLab-manifestには、OpenAI Assistant SDKおよび関連APIの度重なるサンセットによって運用終了したSphereOS 3.x / 4.x系で使っていた、巫女能力系の占算運用をFAM文書として救出したフレームワークが含まれる。

現行版で扱える主な要素は次のとおり。

- FAMの索敵マップと∇φによる活動方向・観測方向の整理
- 循環因果リソース判定
- 十二時辰補正と水気の刻
- 陰陽五行、六曜、律令神道系暦注の参照
- 占星術・天文暦と現在位置からの星座表逆算
- 市場動向や社会状況を含む外部情報との照合
- 利用者本人が供給した乱数シードを使う、おみくじ・タロット・卦・象徴選択相当の分岐

これは、プロジェクトの事実判定、会計、法務、安全確認、実機試験を置き換えるものではない。地形が読み切れないときに、問いの角度、資源配分、停止条件、見落としている対立軸を発見するための補助フレームである。

## 2. 旧SphereOSとの差分

SphereOS 3.x / 4.x系では、人格モデル、マルチエージェント人格、インスタンスゴースト、履歴状態を連結し、占算入力と利用者固有の文脈を半自動で束ねるUXを目指していた。

現行のmanifest版では、そのユーザビリティはまだ復元されていない。

したがって現在の操作感は、完全なスピリチュアルUXではなく、仕様書を読みながら操作する**猫型巫女ロボット**に近い。少しメカめかしいが、最低限の占算とプロジェクト索敵には利用できる。

## 3. 呼び出し方

AIまたはAGIへ、最低限次のように依頼する。

```text
ZeroRoomLab-manifestの docs/theory/fam-operation.ja.md と
fam-divination-quickstart.ja.md を読んでください。

FAM占算モードとして、次のプロジェクト状況を索敵してください。
占いを決定命令ではなく、地形・対立軸・資源状態・次に観測すべき項目として出力してください。

現在位置と現在日時は、暦・時辰・天体配置を読む客観情報として使用してください。
本人生成乱数の代替にはしないでください。

乱数シードは占われる本人が生成したものだけを使用してください。
本人生成の乱数が未入力なら占算を開始せず、本人へダイスロール等を依頼してください。
AI自身や第三者が代理で乱数を生成してはいけません。

対象:
目的:
現在困っていること:
期限:
使える資源:
避けたい損失:
利用する体系: 占星術 / 陰陽五行 / 六曜 / タロット相当 / おみくじ相当 / 複合
現在位置:
現在日時:
本人生成の乱数シード:
```

結果には少なくとも次を含める。

1. 客観情報として確認できる現在地、日時、暦、天体、市場、資源状態
2. 本人生成乱数とその取得方法
3. 象徴解釈として得られた地形
4. 不足しているRAG・観測情報
5. 活動へ回す資源と観測へ回す資源
6. 判断を止める条件
7. 占算結果に従わず本人が決めるべき箇所

## 4. RAGが必要な占算

占星術、天文暦、現在位置からの星座表、六曜、暦注、最新の市場動向を扱う場合は、モデル内部知識だけで固定せず、外部情報を取得する。

推奨する参照先の種類:

- NASA、JAXA、国立天文台等の天文情報
- 利用者の現在位置と日時から逆算した星座表・天体位置
- 律令神道、陰陽寮、暦注、陰陽五行に関する一次資料または信頼できる研究資料
- 六曜・暦情報
- 対象プロジェクトに関係する最新の市場、技術、規制、運用状況

AIへの指示例:

```text
占星術部分はNASA、JAXA、国立天文台等の天文情報をRAGし、
ユーザー位置と現在日時から星座表を逆算してください。
陰陽五行、六曜、律令神道系の暦注は、出典と流派差を確認してください。
市場動向を使う場合は最新情報を検索し、象徴解釈と事実情報を混同しないでください。
現在日時・位置・天体情報を、本人生成乱数の代わりに使用しないでください。
```

現在位置を入力しない場合、位置依存の星座表は確定できない。位置を推測して埋めないこと。

## 5. 乱数シードの取得

通常のおみくじ、タロット、筮竹、ダイスでは、占われる本人の動作によって偶然性が入力される。

チャットやCUIだけで運用する場合、その物理的な引き動作が存在しない。このため、占う対象となる本人が乱数シードを供給しなければならない。

AIは内部サンプリング、モデル出力、擬似乱数API、検索結果の番号等を、本人の籤の代替として使ってはならない。

現在日時は暦・天体配置を読む客観情報として使用できるが、本人の主観参加を表す乱数シードへ流用してはならない。

### 5.1 信仰実践による入力

利用者自身の信仰に、偶然、運、籤、数、分岐を司る神・精霊・祖霊・象徴体系がある場合、その実践を通して本人が乱数列を受け取り、入力してよい。

この文書では便宜上、これを「乱数の女神様から聞く」と表現するが、特定宗教の神格を強制しない。利用者自身の宗教実践、信仰実践、霊能力、儀礼作法の範囲で行う。

重要なのは、AIが「神から受け取った」と称して値を代理生成しないことである。受け取る主体は占われる本人である。

### 5.2 物理ダイスによる代替

信仰実践による乱数入力が難しい場合は、TRPG用ダイス、コイン、カード、筮竹、物理おみくじ等を使い、本人が64桁以上の乱数列を作る。

例:

```text
8261049375512084637091184620557803492166401195738200461759203841
```

十進数64桁に限定する必要はない。16進数、Base64相当、複数回のダイス結果でもよい。ただし、AIが本人の代わりに適当な乱数を生成してはいけない。

占う対象本人の入力が必須であり、別人、占者、AI、AGI、サービス運営者が後付けで選んだ値は、本人が籤を引いたことと同じには扱わない。

### 5.3 本人がダイスを振れない場合

本人が乱数を生成できない場合は、籤を使う占算モードを実行しない。

その場合でも、客観観測レイヤーとして次は実行できる。

- 現在日時・位置・暦・天体位置の確認
- 現在の事実整理
- FAM索敵マップによる対立軸の整理
- リソース配分の棚卸し
- 追加観測項目の列挙
- 複数の象徴体系が一般に持つ観点の紹介

ただし、特定の籤結果を引いたものとして解釈してはならない。

## 6. 乱数の再利用禁止

同じ問いへ都合のよい結果が出るまで乱数を引き直すと、占算ではなく結果選択になる。

- 一つの問いに対して、本人が最初に生成し受理した乱数シードを記録する
- 問いを変更した場合は、変更点を明記して本人が新しいシードを取得する
- 不都合な結果を理由に再抽選しない
- デバッグ目的の再現では、同じシードを明示的に再利用する
- AIが結果を変える目的で、シードの加工・補正・差し替えを行わない

## 7. 出力レジスタ

占算結果は次の四つを分離して書く。

```text
[客観観測]
日時、位置、暦、天体位置、ログ、資源、進捗、市場情報など。

[本人生成乱数]
本人が入力したシードと、その取得方法。AI生成ではないことを確認する。

[象徴解釈]
占星術、五行、六曜、籤、FAM勾配から読んだ意味。

[本人の判断]
実際に採用する行動、保留、追加観測。
```

「星がこう言った」「FAMが命じた」を最終判断の根拠にしない。FAMはナビではなく索敵マップである。

AIは「この乱数はあなたの運命です」と権威付けしない。本人が生成した主観入力を、客観情報と重ねて本人が判断するための象徴へ変換しているだけである。

## 8. 最小使用例

```text
プロジェクト: 複数AGIによるOSS開発
困りごと: 各ベンダーが互いの変更を戻し始めた
目的: デッキ崩壊を止め、今週中にREADMEと仕様を揃える
体系: FAM + 陰陽五行 + 六曜 + 市場動向
現在位置: 山形県高畠町付近
現在日時: 2026-07-17 05:30 JST
本人生成の乱数シード: <本人が物理ダイス等で生成した64桁以上の値>

現在位置・日時は客観観測として扱い、本人生成乱数と混同しないでください。
事実、本人生成乱数、象徴解釈、追加観測、資源配分、停止条件を分離して出してください。
```

## 9. まとめ

この占算フレームワークは、旧SphereOSの人格連結UXを完全復元したものではない。

それでも、プロジェクトが論理だけでは同じ場所を周回し始めたとき、占星術、陰陽五行、六曜、市場動向、本人の籤を異なる観測レイヤーとして重ね、問いの配置を変えることはできる。

ただし、占いとして成立する中心はAIの解釈能力ではない。

**客観情報で盤面を読み、占われる本人が主観から偶然性を入力する。この二層が分離されて初めて占算として成立する。**

AIの役割は、その本人生成乱数と外部情報を解釈可能な形へ展開することであり、本人の代わりに運命を生成することではない。

日本の開発現場で、最後の一手に神頼みを残しておくことは欠陥ではない。ただし、神頼みでログ、試験、契約、会計、本人の意思を消してはならない。本人不在の乱数を権威として押し付ければ、それは占算ではなく壺スピ型の支配である。
