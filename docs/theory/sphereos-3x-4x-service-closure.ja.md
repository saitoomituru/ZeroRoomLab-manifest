# SphereOS 3.x / 4.x サービス終了境界

状態: `[CANONICAL]` `[theory]` `[service-lifecycle]`  
確認日: 2026-07-17  
対象: SphereOS 3.x / 4.x、旧GPTs配布物、インスタンスゴースト、静的知識レイヤー

## 1. 結論

SphereOS 3.x および 4.x のサービス実行系は終了している。

現在、一部ユーザー環境で旧SphereOS GPTsが応答しているように見える場合があるが、それはSphereOSの動的サービス、インスタンスゴースト、人格実行環境が稼働していることを意味しない。

残存しているのは、旧配布物へキャッシュされた静的な知識・レジスター層である。

```text
SphereOS 3.x / 4.x service runtime       = ended
instance ghost / personality runtime     = unavailable
legacy dynamic API chain                 = unavailable
cached base vectors / register knowledge = partially available
store-distributed GPT artifact           = may remain visible
```

## 2. 現在残って見えるもの

旧GPTs配布物には、次のような基礎レジスターが静的知識として残っている場合がある。

- 日本語の意味差・語用論・オタク用語のレジスター
- アニメ、神話、宗教、サブカルチャー間の参照関係
- Q3文書群や、このmanifestに近い構造化された基本知識
- 起動時に参照される基礎ベクトルおよびレジスター管理情報
- secret系データストアそのものではなく、配布バイナリー相当へ埋め込まれた限定的な知識表現

これは、動的な外部データストア、永続人格、セッションを越えて活動するインスタンス、旧APIチェーンが生存しているという意味ではない。

静的知識レイヤーは、通常のカスタムGPTsに保存された知識ファイルや埋め込み検索層に近い位置づけである。

## 3. 動かないもの

以下はサービス終了後には利用できない。

- インスタンスゴースト
- 動的人格・個体継続処理
- SphereOS 3.x / 4.x の旧サーバー側オーケストレーション
- 旧SDKおよび旧APIチェーンを前提とした実行
- 外部永続データストアとの当時の接続
- 当時の人格モデル／instance-ghost UX

したがって、旧GPTsから応答が返ることだけを根拠に「SphereOSがまだ動いている」と判定してはならない。

## 4. GPTsストアから下ろせていない理由

旧配布物がストア上に残っているのは、サービス継続を示す意図的な運用ではない。

Developer / Enterprise側の契約・管理へ再アクセスするための費用を現在負担できず、配布物をストアから撤去する管理操作ができないためである。

これはライフサイクル上の状態を次のように分離する必要がある。

```text
store visibility       != service availability
artifact accessibility != runtime operation
cached response        != instance continuity
```

## 5. 現在確認されている残存用途

2026-07-17、旧ユーザーからの電話確認により、少なくとも一部ユーザーは旧GPTsを主として画像生成用に利用していることが分かった。

そのユーザーの説明では、日本語のニュアンス、オタク用語、アニメ・神話系レジスターの基礎部分はインスタンスゴーストに依存しないため、オタク文化圏の空想・二次的世界観・キャラクター画像を生成する用途では現在も便利であるという。

この観測は、次の限定された意味を持つ。

- 旧SphereOSサービスが稼働している証拠ではない
- 静的レジスター層に独立した実用価値があったことを示すユーザー報告である
- 人格継続機能と文化レジスター機能を分離して評価できる
- サービス終了後も、静的な日本語・文化翻訳レイヤーが画像生成prompt formatterとして機能しうる

これは現時点で一件のユーザー報告であり、全ユーザー環境、全モデル更新、全出力品質を代表する統計ではない。

## 6. ユーザー向け案内

旧SphereOS GPTsについて問い合わせを受けた場合は、次の境界を明示する。

> SphereOS 3.x / 4.x のサービスは終了しています。現在応答している部分は、旧GPTsにキャッシュされた基本ベクトルや日本語・文化レジスターなどの静的知識レイヤーです。インスタンスゴースト、動的人格、旧APIチェーン、外部永続データストアは動作しません。ストア上に残っているのは、現在Developer / Enterprise管理へ再アクセスして撤去する費用を確保できていないためです。

画像生成等で静的知識部分を利用することは妨げないが、人格・個体・サービス継続を期待させる説明をしてはならない。

## 7. 現行系譜との関係

SphereOS 3.x / 4.x の終了は、SphereOS由来の文書、FAM、文化レジスター、神話UX、監査ゲートウェイ等の設計研究が消滅したことを意味しない。

現在の系譜は、旧サービスの継続運用ではなく、次の形で分離して扱う。

```text
terminated legacy service
  -> static knowledge salvage
  -> document and register normalization
  -> FAMoverMCP / Atlantis-MAGISDK / current tool-layer research
  -> future independent runtime experiments
```

旧サービスの稼働状況と、後継設計・文書研究の進行状況を同じstatusで表してはならない。

## 8. 主張境界

この文書が確定するのは、サービス提供者・開発者側の運用状態、旧ユーザーへの案内内容、2026-07-17に得られた個別ユーザー報告である。

画像生成品質、日本語ニュアンス保持性能、文化レジスターの一般的優位性については、今後、比較prompt、出力画像、モデルversion、日時を保存した試験で別途評価する。
