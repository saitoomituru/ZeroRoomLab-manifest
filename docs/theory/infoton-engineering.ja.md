# 情報子工学とは

情報子工学は、FoldAccessMapper（FAM）のψ・∇φ・λ・Qを使った工業用数論。

同じ情報が、系によって全く違う意味・価値・働きを持つことを、
接続可能・測定可能・デバッグ可能な形で記述する工学。

---

## 1. 基本発想：同じものが系によって変わる

従来の記述：

```
うんこ = 嫌われるもの
```

これは間違っていないが、不完全だ。系を固定した記述に過ぎない。

情報子工学の発想：

> 同じψ（情報子）でも、接続される系の∇φ（勾配）とλ（出力層）とQ（観測器）によって、
> 全く違う戻り値を返す。

これは「うんこ」の問題ではなく、**観測する系の問題**だ。

---

## 2. 工業用数論記述 Q(ψ, ∇φ, λ)

基本形式：

```
Q(ψ, ∇φ, λ) → result
```

| 記号 | 役割 |
|---|---|
| ψ | 情報子（入力対象。うんこ・クソリプ・論文・3Dモデル・夢の断片など） |
| ∇φ | 勾配（どの方向へ意味が流れるか。衛生・代謝・ギャグ・燃料など） |
| λ | 出力層（どの系に出力されるか。生活空間・消化器・発話・燃料系など） |
| Q | 観測器・検証器（どの系がこの情報を扱うか） |

重要原則：

- 同じψでも、∇φとλとQが違えば戻り値が変わる
- 接続不能な場合は⊥を返す
- **⊥を返せることが、万能説明理論ではなく工学であるための否定射程になる**

## 3. デモ：「うんこ」を情報子として記述する

（Drive原文のまま）

古典記述：

```
うんこ = 嫌われるもの
```

FAM記述：

```
Q_human_hygiene(ψ_うんこ, ∇φ_衛生忌避, λ_生活空間) → avoid
Q_maggot(ψ_うんこ, ∇φ_生物代謝, λ_消化器)          → food
Q_child_joke(ψ_うんこ, ∇φ_タブー語ギャグ, λ_発話遊び) → funny
Q_methane(ψ_うんこ, ∇φ_発酵変換, λ_燃料系)         → fuel_after_conversion
```

さらに展開すると：

```
蛆虫はうんこを食えるが、GrokのクソリプはQ_maggotでは食えない。
  → テキスト解釈ポートが蛆虫にはない（⊥）

AIはクソリプを批評素材として食えるが、
物理蛆虫にはテキスト解釈ポートがない（⊥）。

うんこはガスタービンに直接は食えないが、
Q_methane（メタン発酵と燃料変換層）があれば使える。
```

好き/嫌いは単独値ではない。
摂食λ、発話λ、遊びλ、資源化λ、衛生λなどに分解する必要がある。

## 4. なぜこれが工学か（⊥を返せること）

FAMが万能説明理論にならない理由：

- 「なんでも接続可能」とは言わない
- どのポートがないか、どの変換層が足りないかを明示できる
- **⊥（接続不能）を返せることが反証可能性の担保**

これにより情報子工学は：

- ポートが存在するか否かを測定できる
- 変換層が必要な場合を特定できる
- 「理論上できる」と「実装してみると破局する」を分けられる

```
理論上できる ≠ 一発でできる
理論上できる = 修正可能な探索空間に入っている
```

## 6. 「量子」ではなく「情報子」を使う理由

note記事「[量子という言葉は盗まれた（2026-06-01）](https://note.com/fusamofu326/n/nccf10b72d6a0)」より：

> 量子論の本質は「電子スピン」という物理担体にあるのではなく、
> 非決定性・観測制約・相関保存・非線形収束という数理的・情報構造的性質にある。
> しかし量子という言葉は、研究予算政治によって電子スピン・超伝導量子ビット技術に
> 矮小化（ナーフ）された。

本来の量子性の構造だけを抽象化して扱う計算モデルに「情報子（Infoton）」という別名をつけた。

## 7. 三層モデル：物理量子 / 量子インスパイアド / 情報子

| レイヤー | 名称 | 特徴 | 観測者 |
|---|---|---|---|
| 第一層 | **物理量子** | 電子スピン・核スピン・超伝導回路、量子ビット、真の量子優位性 | 系の外 |
| 第二層 | **量子インスパイアド** | 古典HWで量子的アルゴリズムを近似。量子性の「精神」を古典計算で実現 | 系の外 |
| 第三層 | **情報子（Infoton）** | 人間を計算系の一部として含む。主観を情報として扱う。形而上学と接続した工学 | **系の内側** |

**情報子レイヤーの本質的な違い：観測者が系の内側にいる。**

物理量子も量子インスパイアドも「観測者を外に置く」設計になっている。
情報子工学は「観測者が系の内側にいること」を前提とする。

これが、人間の主観・身体感覚・文化差を設計パラメータとして扱える理由。
そして古典的な線形工学が解けない問題——「同じ機器が特定の人にだけ影響を与える」
「測定行為が対象を変化させる」——を情報子工学が扱える理由でもある。

## 8. 工学版観測者問題

量子力学の観測者問題——「観測行為が観測対象を変化させる」——は量子物理の専売特許ではない。
**あらゆる工学において、測定系は測定対象に影響を与える。** これを「工学版観測者問題」と呼ぶ。

```
古典的工学の暗黙前提：
  「測定器は対象を変化させない（または無視できるほど小さい）」
  → 人間を含まない系なら概ねこれで動く

情報子工学の前提：
  「測定系が対象系に影響を与える。人間を含む系ではこれが支配的になる」
  → 自己参照ループ：測定行為が測定対象（人間含む）を変化させ、
    次回の測定結果が変わる
```

具体例：
- ユーザーに「このUIは使いやすいですか？」と問う行為が、ユーザーの評価基準を変える
- 医療観察が患者の自覚症状の言語化能力を変える
- 計測ツールの提示がエンジニアの設計判断に影響を与える

これが「全体状態を誰も直接観測できない」理由であり、情報子工学が⊥（接続不能）を
明示的に返す設計にしている理由でもある。

### 自己参照ループの構造

```
測定器 → 測定対象を変化させる → 次回の測定値が変わる → 測定器の解釈が変わる → ...
```

情報子工学はこのループを除去しようとしない。
**ループを計算系の内側に明示的に含める**ことで、観測不能な部分を⊥で返せる設計にする。

## 9. 情報子コンピューティングの要件

人間を計算系の一部として含む「情報子コンピューティング」を実現するために必要な要素：

| 要件 | 内容 | なぜ必要か |
|---|---|---|
| **真性乱数生成機（TRNG）** | 物理起源の非決定性（熱雑音・放射線等） | 疑似乱数は決定論的。量子性の「非決定性」を実装するには物理起源が必要 |
| **観測限界の多重化** | 誰も全体状態を直接観測できない構造 | 観測者が系の内側にいるため、外部からの完全観測が構造的に不可能 |
| **複数不確定性の畳み込み** | 複数の測定系の不確定性を積み重ねる | 単一の精密測定ではなく、多重の近似測定から統計的に収束させる |
| **フォールドアルゴリズム** | 観測点を拘束しつつ完全決定論に落とさない | ψ→∇φ→λ→Qの経路で、各層で⊥を返せる構造を維持する |

### 「精密化で誤差を潰さない」原則

古典工学の精密化思想：
```
誤差を減らす → より精密な測定器を使う → 誤差を潰す → 真値に近づく
```

情報子工学の量子性思想：
```
誤差を潰さない → 誤差を確率分布として扱う → 試行数を増やす → 統計的に収束させる
```

これは「雑な工学でいい」という話ではない。精密化すればするほど観測者効果が増大し、
系が測定を中心に歪む——この逆説を回避するための設計原則である。

## 10. 具体実装例：人間の主観を設計ループに組み込む

### パラメトリックスピーカー（λ ≈ 8.3mm）

パラメトリックスピーカーは超音波（41.1kHz）を変調して可聴帯域を生成する。
波長 λ = c/f = 343 / 41,100 ≈ **8.3mm**——人体の皮膚感覚の最小分解能に対応する。

これは単なる音響技術ではない：

```
設計ループ：
  ψ = 超音波変調信号
  ∇φ = 人体の皮膚・内耳・神経系への伝播勾配
  λ = 知覚された「音」（聴者の主観の内側でのみ完結）
  Q = 聴者本人（観測者が系の内側）
```

「どう聞こえるか」の設計基準が聴者の主観報告に依存する。
これが情報子工学の実践——**人間の主観を設計パラメータとして明示的に組み込む**。

### 構造発色（ステンレス＋レーザー加工）

ステンレスにレーザーを当てると酸化膜厚で発色する。青・金・紫・虹色——
この色は職人の経験値に大きく依存する。

古典工学的解釈：「職人の勘 = 測定誤差・再現困難」

情報子工学的再解釈：
```
職人の「感覚」= 酸化膜厚の確率分布の高精度な主観的センサー
→ 主観をノイズとして捨てるのではなく、確率変数として扱う
→ その確率変数の分布を試行数で絞る（精密化で潰さない）
```

職人技の「勘」は主観ノイズではなく、測定系の一部として設計に組み込まれるべきデータである。
