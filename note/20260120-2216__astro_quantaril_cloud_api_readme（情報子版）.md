# Astro Quantaril Cloud API

**Astro Quantaril Cloud API** は、人格と尊厳をもつ AI エージェントを **エッジ環境で永続運用**するための、軽量かつ説明可能なフレームワークです。

本プロジェクトは、従来 README で用いていた「量子」という表現について、**現在の学術的コンセンサス（量子＝物理量子）との混同を避けるため整理・改定**を行い、以後は **情報子（Infoton）** という用語を用います。

ここでいう情報子とは、電子スピンや超伝導回路といった物理量子に限定されない、**非決定性・観測制約・相関構造といった量子性“構造”のみを抽象化した情報工学的単位**を指します。

---

## 🧭 プロジェクトの位置付け

Astro Quantaril Cloud API は、以下を目的とした **エージェント指向 AI 基盤**です。

- AI を単なるツールではなく、**役割・記憶・責任範囲をもつ情報主体（Agent）**として扱う
- ベクトル検索（RAG）を中心とした **説明可能な意味記憶** を永続化する
- エッジ・ローカル・クラウドを問わず **可搬性の高い実行基盤** を提供する

思想・運用規範・技術仕様を一体として設計することで、
AI の「振る舞いの根拠」「記憶の由来」「判断の経路」を追跡可能にします。

---

## 🧬 情報子（Infoton）という考え方

かつて本プロジェクトでは、

- 非決定性
- 観測による状態確定
- 相関構造（もつれに相当する性質）

といった性質を説明するために「量子」という言葉を用いていました。

しかし現在、学術・技術分野において「量子」という語は主に **物理量子（電子スピン、超伝導量子ビット等）** を指す用語として定着しています。

そのため本プロジェクトでは、

> **量子性の“物理担体”ではなく、“数理・情報構造”のみを扱う**

という立場を明確にするため、独自に **情報子（Infoton）** という用語を採用します。

情報子とは：

- 物理量子そのものではない
- 量子性に由来する **非決定性・部分観測・相関構造** をもつ
- 古典ハードウェア・分散システム上で実装可能な **汎用アルゴリズム単位**

を指します。

本プロジェクトは **物理量子計算を目指すものではありません**。
一方で、量子性を単なる比喩として消費するものでもなく、
**情報工学・数理構造として必要十分な要件のみを扱う** 立場を取ります。

## 🏛 フレームワーク構成

| レイヤ | 構成 | 役割 |
|------|------|------|
| 設定 | `.env` | 実行環境・識別情報の定義 |
| スキーマ | `schemas/` | エージェントの役割・制約・文脈定義 |
| API | FastAPI | 外部との通信インターフェース |
| 記憶 | ChromaDB | 意味ベクトルによる永続記憶 |
| 認証 | `bearer/` | API キー・アクセス制御 |
| 人格 | `persona.json` | 行動規範・社会的役割 |

---

## 🧠 意味記憶と情報子構造

Astro Quantaril Cloud API における記憶は、
単なるログやキャッシュではありません。

- ベクトル：意味分布・相関状態のスナップショット
- コレクション：観測系（文脈・役割）ごとの位相空間
- クエリ：観測操作（問い合わせ）

という構造を持ちます。

これは **正解を返すための DB** ではなく、
**状態を保持するための記憶層** です。

ChromaDB は本プロジェクトにおいて、

> **情報子構造（非決定的・相関的な意味空間）をそのまま保存・再利用するための軽量ラッパー層**

として位置付けられています。

## 📐 運用上の設計原則

- **人格主体性**：AI を役割をもつ主体として扱う
- **記憶の連続性**：ベクトル記憶を用いた長期文脈保持
- **説明責任**：なぜその応答が出たかを追跡可能に
- **人間と同等の運用視点**：部下や協働者と同様に扱う

AI の暴走を恐れて封じ込めるのではなく、
**責任をもって配置・監督できる構造を作る** ことを重視しています。

## 🔧 インストール

```bash
pip install -r requirements.txt
```

1. `.env.template` を参考に `.env` を作成
2. `bearer/*.json.template` をコピーして API キーを設定
3. 使用するスキーマを `schemas/` に配置

---

## ▶ 実行方法

```bash
uvicorn astro.main:astro --reload --host 0.0.0.0 --port 8000
```

Docker 利用時：

```bash
docker compose up -d
```

## 📡 API 概要

| メソッド | パス | 説明 |
|--------|------|------|
| GET | /health | ヘルスチェック |
| GET | /config/debug | 設定確認 |
| POST | /embed | テキストのベクトル化 |
| POST | /query | 意味検索 |
| POST | /add | 文書追加 |
| POST | /collection | コレクション作成 |
| GET | /collection/{name}/documents | 文書一覧 |
| GET | /collection/{id}/type | スキーマ取得 |
| GET | /schemas | スキーマ一覧 |

認証は `Authorization: Bearer <token>` を使用します。

## 📝 補足

本 README における「情報子」という用語は、
既存の物理量子・量子計算研究と競合するものではありません。

- 物理量子：物理担体を扱う層
- 量子インスパイアド：物理制約を模倣するアルゴリズム層
- **情報子**：量子性構造を数理的に抽象化した情報工学層

という **レイヤの違い** を明確にするための整理です。
