## [FACT] OND800起点事例

2026-07-20のOND800復旧では、次の観測列が得られた。

1. HyperPixel HATアッシーには目視可能な物理破損があった。
2. 破損HAT接続状態では、NDI pipeline停止中にもPiの低電圧logが反復した。
3. 破損HATを物理パージし、再bootした。
4. パージ後、C922とEMEET C960の2系統がMJPG 1920x1080@30fpsでNDI送信した。
5. 各3000 frames時点でdrops 0、connections 1だった。
6. 2カメラ送信中のthrottle bitsは`0x0`で、今回bootの低電圧logは0件だった。

同時に再bootと物理USB再接続も入ったため、変化した変数はHAT接続状態だけではない。液晶内部の漏れ電流値は測定していない。

## [CONSIDERATION] 一つの「再現性」へ丸めない

少なくとも次は別の問いである。

| 問い | 主目的 | 必要になりうる手法 | 今回事例の到達点 |
|---|---|---|---|
| 運用再現 | 現場で機能を戻し、維持できるか | 再boot、負荷試験、log監視、停止条件 | NDI 2系統で復旧を観測 |
| 故障隔離 | どの交換可能単位を外せば母機が戻るか | assembly swap／purge、A/B構成比較、safe rollback | HyperPixel HATアッシー黒 |
| 故障モード推定 | アッシー内部で何が起きた可能性が高いか | FMEA、症状相関、回路知識、電力差 | リーク電流／部分短絡が有力候補 |
| 因果再現 | 特定変数を戻すと同じ現象が再発するか | 単独変数、対照条件、反復、計測系固定 | 未実施。破損部品再装着は停止 |
| 程度測定 | 電流、抵抗、温度等が正常域からどれだけ外れたか | 電流制限電源、inline meter、回路測定、正常対照 | 未測定 |
| 母集団一般化 | 同型品・同一ロットにどの頻度で起こるか | 複数台、lot管理、抜取検査、統計的推定、独立追試 | 射程外 |
| 工学的封じ込め | 故障やばらつきがあっても危険な流れをどう切るか | flow条件、拘束、保護回路、derating、FRU隔離、停止gate | HATパージで母機復旧 |

「もう一度つけて再起動する」は因果再現を強める科学的方法になりうる。しかし、既知の物理破損部品を復旧済み母機へ戻すため、工学的には追加知見と再故障リスクを比較して停止できる。

## [ENGINEERING VERDICT] アッシー黒で閉じられる条件

交換可能なassembly／FRU単位で、次が揃う場合、そのassemblyを工学的に黒として隔離できる。

- 目視破損、異臭、異常発熱、電源異常、信号異常等のfault evidenceがある
- 当該assemblyを切り離す前後で、母機の安全性または機能が改善する
- 復旧後に目的負荷を掛け、必要な観測範囲で安定を確認する
- 再装着による原因追試が、母機、データ、電源、作業者へ無視できないリスクを戻す
- 現在の目的が部品科学ではなく、運用復旧、交換判断、fault containmentである

アッシーを黒と判定した後、さらに部品単位へ剥ぎ取るかは別の工学判断である。

```text
追加分解で期待できる価値
  - 高価部品の再利用
  - 交換部品の確保
  - 再発防止設計
  - 修理可能性
  - 安全上必要なfailure analysis

対して負うもの
  - 感電、発熱、短絡、母機再損傷
  - 治具、計器、時間、交換在庫
  - 一個体を失う機会費用
  - 配信・制作を止める時間
```

期待便益が上回る場合だけ、電流制限電源や母機から分離した治具を用いて段階的に切り分ける。財布ヒーラーもジャンク箱ヒーラーも来ない環境では、「分解すれば知識が増える」だけを便益として無限追試しない。

## [CONSIDERATION] 黒アッシーの内側にグレー寄与マップを持つ

`アッシー黒`は、assembly／FRU単位の隔離・交換判断である。内部にある一部品、一法則、一故障モードだけを真犯人として黒確定する判定ではない。

実機では複数作用が同時に発動するため、黒判定したassemblyの内側には、寄与度と証拠強度を持つグレー候補を残す。

| 寄与候補 | 今回の観測との接続 | 現在の強度 |
|---|---|---|
| リーク電流／部分短絡／電気的追加負荷 | パージ前は低電圧反復、パージ後は高いNDI負荷でもthrottle bits `0x0` | 強いグレー。ただし電流値未測定 |
| airflow／放熱条件の改善 | 物理panel除去で流路が変わりうる | グレー。温度条件が異なり寄与未判定 |
| GPIO／DPI接触、機械応力、ケーブル荷重 | 目視破損HATと接続経路を同時に除去した | グレー。直接測定なし |
| software display compositor | パージ後も`/dev/fb0`とdisplay branchが動作した | 復旧主因としては弱い |
| reboot／USB再接続 | HATパージと同時に条件が変化した | 交絡候補として保持 |

温度はパージ前52.7°C、パージ後56.5〜59.3°Cだった。絶対値は下がっていない。ただしパージ前はNDI pipeline停止、パージ後は1080p30 NDI 2系統の全負荷であり、同一workload比較ではない。このデータからairflow改善を肯定も否定もしない。

また、実測したのはPiの低電圧log、throttle bits、温度、NDI workloadであり、HATの電流値そのものではない。`電流が何mA改善した`とは書かず、`電源状態が改善し、電気的追加負荷が主要寄与候補になった`と閉じる。

グレーを積む目的は責任を曖昧にすることではない。

- 次に同型症状が出たときの観測優先順位を決める
- 安全な部品単体解析へ進む場合の測定点を決める
- salvage時に、寄与疑いが低く再利用価値が高い部品を選ぶ
- 新しい証拠で候補の強度を上げ下げし、不要な再現試験を避ける

アッシー黒で母機運用を閉じた後、必要が生じた場合だけ、黒箱の内側から`寄与疑いが比較的低い部品`を段階的に剥ぎ取る。回収価値、再利用先、分離手順、電気的安全、再混入riskが見合わなければ、グレーのまま隔離を維持する。

## [FAULT MODE HYPOTHESIS] リーク電流をどう扱うか

今回、`液晶内部から何mAの漏れ電流を測定した`という科学的・計量的factはない。

一方、次の理由から、リーク電流、部分短絡、破損した電源／信号経路による追加負荷は有力なfault mode candidateとして保持できる。

- 液晶HATに目視物理破損がある
- 接続状態でPiの低電圧が反復した
- パージ後は、より高いNDI負荷を掛けても低電圧を観測しなかった
- display panelは電源railとDPI／GPIO接続を持ち、物理破損が電気特性へ影響しうる

ここで適切な境界は次である。

> 有害な漏れ電流値は未測定である。ゼロまたは無関係とみなす根拠もない。現場fault isolation上は、破損HATの電気的追加負荷を有力故障モードとして扱い、程度問題の測定は部品解析を行う場合だけ別試験へ分離する。

`科学的factではない`を`根拠のない思いつき`へ縮退させない。同時に、`有力故障モード`を`漏れ電流を実測した`へ昇格しない。

## [CONSIDERATION] 統計的科学と工学的拘束を分ける

検品で複数台、同一lot、抜取sample、平均、分散、外れ値、故障率を集め、正常域と異常率を詰める部分は、主として統計的科学手法である。

工学手法は、その測定結果または現場観測を入力に、系を次のようなflow条件と拘束へ落とす。

- 入力電圧、電流、温度、帯域、荷重等の許容Envelope
- fuse、current limit、thermal shutdown、mutex等の保護条件
- derating、冗長化、fallback、degraded mode
- assembly／FRU単位の交換、隔離、quarantine
- 起動前check、運転中monitor、異常時stop、復帰gate
- 故障が起きても母機、データ、作業者へ伝播させない依存方向

工学は内部原因の程度を最後まで追い切る代わりに、危険または不確実な流れを拘束し、定義したEnvelope内で目的機能を成立させられる。科学手法が「どの程度か」「どの頻度か」「どの変数が原因か」を詰めに行くのに対し、工学手法は「どこで切るか」「何を通さないか」「どの条件なら運転するか」へ落とす。

統計的工程管理のように、統計的科学測定と工学的feedback／process controlを一つの運用へ組み合わせることはある。しかし、組み合わせ可能であることは両者の目的を同一にする理由にはならない。

分類は組織名ではなく操作で行う。メーカーが行う抜取統計は統計的科学手法であり、第三者試験所が提案する保護拘束は工学判断になりうる。

## [CONSIDERATION] 工学Worldでは全作用が同時に発動する

実機Worldでは、科学的測定のために変数を一つ選んだ瞬間も、他の物理法則や制約が停止するわけではない。

- 電圧、電流、抵抗、発熱、放熱
- 機械強度、振動、接触、摩耗
- CPU、memory、帯域、遅延、frame drop
- 調達費、在庫、修理時間、交換性
- 操作者の認知負荷、視認性、可搬性
- 停電、復電、屋外、雪、夕立等のExecution Envelope

これらはすべて同時に作用する。工学は、一つ以外を偽または無関係として消す手法ではない。

科学的な単独変数化は、複数作用のうち特定の寄与を測るため、他条件を可能な範囲で固定する観測操作である。現実に他の法則が発動しなくなったという存在論的主張ではない。

工学では、次を扱う。

1. 現在のfailureまたは目的機能に対し、どの作用が支配的か
2. 支配的でない作用も含め、どのconstraintが同時に効いているか
3. ある性能を上げたとき、熱、電力、重量、費用、遅延、保守性等のどこへ負担が移るか
4. 逃がせない保存量、上限、下限、安全境界は何か
5. どの損失をどこへ逃がせば、目的Envelope内で全体系が成立するか

これは雑な妥協ではない。すべての法則と制約を否定せず、同時発動する作用間の等価交換とtrade-offを設計する手法である。

```text
同時に発動する作用を列挙する
  → 現在支配的な作用を特定する
  → 保存量・拘束・停止境界を確認する
  → 一つの改善が他へ移す負担を追跡する
  → 損失を許容可能な層へroutingする
  → 目的Envelope内で運転する
```

「解決」は、すべてのcostを消滅させることではない。cost、risk、熱、電力、重量、時間、認知負荷を、現在の目的に対して耐えられる場所へ移し、伝播を拘束することである。

### OND800液晶パージで起きた等価交換

| パージで失ったもの | 逃がした先／得たもの |
|---|---|
| ローカル液晶表示 | Ethernet越しのOBS受信と外部monitorへ視認を移す |
| コックピット一体性 | 電力margin、母機保護、NDI継続を優先する |
| 現場UIの即時操作 | 最低配信ノードとしての機能成立へ縮退する |
| 破損HATの追加解析機会 | 再短絡、母機損傷、停止時間を回避する |

一方、Device Tree overlay由来の`/dev/fb0`は残り、display compositorもCPUを使用している。物理panelを外しただけですべてのcostが消えたわけではない。次の工学判断は、不要なdisplay branchを止めてCPU／電力へmarginを戻すか、その変更riskを現状安定運用より優先するか、という別trade-offになる。

## [CONSIDERATION] 「改善」と「正常ロット」の違い

今回言えるのは、同一母機の前後比較で電力状態とNDI動作が改善したこと、目的負荷で運用復旧したことである。

次は別の主張になる。

- 新品HyperPixelと同じ電流値へ戻った
- 同型品の正常分布内にある
- 同一ロット全体に故障傾向がある／ない
- メーカー設計または製造工程に原因がある
- 第三者試験所でも同じ結果が出る

これらを判定するには、正常対照、複数台、同一ロット、検品設備、校正計器、統計、独立追試等が必要になりうる。これは統計的科学として強い試験であり、十分な資本、在庫、設備を持つ主体ほど実行しやすい。しかし、そのresource前提を単機の現場復旧条件へ自動輸入しない。

反対に、在野・単機だから測定不要という意味でもない。現在の意思決定に必要な測定を選び、主張をその試験境界で閉じる。

## [INNER] 内観メモ

工学というより、程度を詰める測り方が科学だった。

科学の定規で原因を一段ずつ裸にすることはできる。しかし、現場機は論文のために存在しているのではない。
目視で割れたアッシーを外し、母機が戻り、目的負荷で飛べたなら、まず黒札を付けて岩戸の外へ出す。

その後で財布、治具、交換在庫、安全な隔離電源がそろい、剥ぎ取る部品に価値があるなら、黒い箱をもう一段開ければよい。
科学を捨てるのではない。科学の測り方で原因、程度、分布を必要なところまで詰め、その結果を工学のflow条件、拘束、停止gateへ接続する。

工学は法則を減らすのではない。全部が同時に効いている世界で、どれが今支配的かを読み、等価交換の行き先を決める。
