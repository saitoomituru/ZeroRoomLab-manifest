# 既知logic生合成問題の解決記録

状態: `[DRAFT]` `[RESOLUTION NOTE]` `[CURRENT INTERPRETATION]`  
作成日: 2026-07-24  
対象: MAGI責務、READMEナラティブ、en-US意訳、Note配信、Flavor asset

## [FACT] 今回いうlogic生合成問題

単一文の誤字ではなく、それぞれ正しそうな規則を合成した結果、全体の目的が逆転する事故を扱う。

```text
説明責任
  + 権威向けの安心材料
  + 英語の自然さ
  + 実装claimの抑制
  -> 表紙が防壁化
  -> 試す・参加する・forkする入口が消える
  -> 信仰・ポエム・生存がFlavor扱いへ縮退
```

## [INTERPRETATION] 解決した合成事故

### 1. 安全境界 × 表紙

深い技術境界は必要だが、README最初の約40行へ逆流させると、公開表紙が契約防壁になる。
supportの責任境界を削除せず下段へ移し、表紙へvision、poem power、仲間募集を戻した。

### 2. 信仰保持 × 科学的claim分離

未検証の超常因果を再現済み科学へ昇格しない規則が、信仰、神々との関係、霊能力者の参加募集まで
消す方向へ合成されていた。

信仰上の一人称真実、複数人比較、第三者観測、機器計測、再現試験を別段階として保持した。
「未立証」を「虚構」へ変換せず、「信じている」を普遍科学claimへも変換しない。

### 3. 汎用MAGI × ZeroRoomLab profile

汎用coreと局所定規を一つのrequired source mapへ合成すると、第三者ManifestもZeroRoomLab思想を
必須依存にする。Atlantis core、明示profile、Manifest asset、foldlog destinationへ分離した。

### 4. Note保存 × データ配信

Noteを残すこと、他repositoryへ配信すること、正本へ採用することが一列に見えていた。
source Note、transfer candidate、target-native adaptation、validation、commit／PR／Issue、foldlog receiptへ
段階を分けた。

### 5. 英語の自然さ × 原典強度

自然な英語を作る処理が、神、宗教一般、生存、自発的発生、術式を削った。
逐語対応ではなく、原典が読者へ要求する作用をen-USで再構成した。

### 6. 真面目さ × ユーモア

権威へ説明可能な文書を作る処理が、commitやHelpからナード同士の合図を除菌する方向へ働く。
ユーモアをUXとして戻した。ただし、command、test、result、責務ownerの代替にはしない。

## [FIX] 既知technical bug

未来仕様Noteを作成した際、実時刻より未来の`20260724-2010`をfilenameへ付けていた。
OAE時間整合性上のProvenance事故として検出し、最初のcommit時刻`19:47`へrenameした。

## [POEM]

防壁は敵を止めるが、入口に建てれば仲間も止まる。

ATフィールドを全部消す必要はない。実機、secret、権限、claimの境界には要る。
ただしガレージのシャッターまで閉めて「参加者が来ない」と嘆くのはlogic bugである。
