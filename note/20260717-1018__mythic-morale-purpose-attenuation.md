# 神話的士気管理と目的関数矮小化連鎖の正本化

状態: `OBSERVED / INFERRED / PLANNED`  
日付: 2026-07-17

## OBSERVED

- ユーザーは、安全を成立させる二つの半分を「ロマンや神話の欠乏」と「ロマンや神話に対して十分な
  安全率を持った資源」と表現した。
- 神話、夢、ロマン、哲学をKPIへ回収せず、やりがい搾取の道具にもさせない境界が要求された。
- 資金、計画、管理、実行を下る間に目的関数が段階的に矮小化され、最後に手抜きとして見えるという
  組織的連鎖が提示された。

## INFERRED

- 意味と資源を独立軸にすると、神話だけが強い状態を「やりがい搾取」、資源だけがある状態を
  「目的関数ドリフト」として区別できる。
- 「事故原因の半分」は実測割合でなく、双方がなければ安全が閉じないというLayer Bの完全性モデルである。

## PLANNED

- PSYCHO-Py800MCPのLayer AへPurpose Function Ledger、PurposeIntegritySignal、
  ResourceIntegritySignal、append-onlyのstage変更記録を写像する。
- 人物の忠誠心や熱意ではなく、operationとsnapshotへsignalを付与する。

## Claim Boundary

事故割合、事故低減効果、組織文化、士気向上は測定していない。特定組織、職種、人物を原因と断定しない。
Layer Bの強い因果仮説と詩的表現を、Layer Aの観測済み事実へ昇格させない。

## 内観メモ

夢だけ渡して足場を抜けば搾取になる。足場だけ作って行き先を消せば、誰も橋を守らなくなる。
ロマンは無料の燃料ではなく、どこまで安全率を積むべきかを伝える設計荷重なのだと思う。
