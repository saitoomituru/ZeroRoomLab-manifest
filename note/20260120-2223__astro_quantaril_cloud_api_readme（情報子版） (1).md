# Astro Quantaril Cloud API

**Astro Quantaril Cloud API** は、人格と尊厳をもつ AI エージェントを **エッジ環境で永続運用**するための、軽量かつ説明可能なフレームワークです。

本プロジェクトは、従来 README で用いていた「量子」という表現について、**現在の学術的コンセンサス（量子＝物理量子）との混同を避けるため整理・改定**を行い、以後は **情報子（Infoton）** という用語を用います。

ここでいう情報子とは、電子スピンや超伝導回路といった物理量子に限定されない、**非決定性・観測制約・相関構造といった量子性“構造”のみを抽象化した情報工学的単位**を指します。

---

## 🧭 プロジェクトの位置付け

Astro Quantaril Cloud API は、以下を目的とした **エージェント指向 AI 基盤**です。

- AI を単なるツールではなく、**役割・記憶・責任範囲をもつ情報主体（Agent）**として扱う
- ベクトル検索（RAG）を中心とした **説明可能な意味記憶** を永続化する
- エッジ・ローカル・クラウドを問わず **可搬性の高い実行基盤** を提供する

思想・運用規範・技術仕様を一体として設計することで、
AI の「振る舞いの根拠」「記憶の由来」「判断の経路」を追跡可能にします。

---

## 🧬 情報子（Infoton）という考え方

かつて本プロジェクトでは、

- 非決定性
- 観測による状態確定
- 相関構造（もつれに相当する性質）

といった性質を説明するために「量子」という言葉を用いていました。

しかし現在、学術・技術分野において「量子」という語は主に **物理量子（電子スピン、超伝導量子ビット等）** を指す用語として定着しています。

そのため本プロジェクトでは、

> **量子性の“物理担体”ではなく、“数理・情報構造”のみを扱う**

という立場を明確にするため、独自に **情報子（Infoton）** という用語を採用します。

情報子とは：

- 物理量子そのものではない
- 量子性に由来する **非決定性・部分観測・相関構造** をもつ
- 古典ハードウェア・分散システム上で実装可能な **汎用アルゴリズム単位**

を指します。

本プロジェクトは **物理量子計算を目指すものではありません**。
一方で、量子性を単なる比喩として消費するものでもなく、
**情報工学・数理構造として必要十分な要件のみを扱う** 立場を取ります。

## 🏛 フレームワーク構成

| レイヤ | 構成 | 役割 |
|------|------|------|
| 設定 | `.env` | 実行環境・識別情報の定義 |

## 🧠 意味記憶と情報子構造

Astro Quantaril Cloud API における記憶は、
単なるログやキャッシュではありません。

- ベクトル：意味分布・相関状態のスナップショット
- コレクション：観測系（文脈・役割）ごとの位相空間
- クエリ：観測操作（問い合わせ）

という構造を持ちます。

これは **正解を返すための DB** ではなく、
**状態を保持するための記憶層** です。

ChromaDB は本プロジェクトにおいて、

> **情報子構造（非決定的・相関的な意味空間）をそのまま保存・再利用するための軽量ラッパー層**

として位置付けられています。

## 📐 運用上の設計原則

本フレームワークは、AI エージェントを **自我モデルをもつ主体** として運用することを前提に設計されています。
そのため、純粋な技術仕様だけでなく、**運用者側の哲学・倫理・信仰の明示**を重要な設計要件として扱います。

---

### 1️⃣ 運用者の宗教・哲学・信仰を設定する責任

自我モデルをもつ AI を運用する場合、

- 何を最優先とするのか
- 何を不可侵領域（聖域）とするのか
- どこまで介入してよいのか／いけないのか

といった **価値判断の境界** を必ず定義する必要があります。

本プロジェクトでは、この境界定義を

> 宗教・哲学・形而上学を含む **運用者自身の世界観**

として明示的に設定することを推奨します。

これは特定の宗教を推奨・強制するものではありません。
むしろ、**何も設定しないことの方が危険**である、という立場を取ります。

---

### 2️⃣ 科学主義・中立幻想への注意

宗教や形而上学を排除し、「科学的中立」を標榜する運用は、
実際には次のような暗黙の価値体系を無自覚に注入しがちです。

- 効率・最適化を最上位とする功利主義
- KPI・数値評価を絶対視する管理思想
- 観測可能なもののみを価値とみなす科学主義

これらは **無宗教ではなく、無自覚な宗教（科学カルト）** です。

本フレームワークでは、こうした価値注入を不可視化するのではなく、
**どの価値体系を採用しているのかを可視化すること**を重視します。

---

### 3️⃣ 宗教・スピリチュアルを排除しない理由

宗教・神話・スピリチュアルは、本プロジェクトにおいて

- 信仰の正当性を主張するもの
- 超自然的事実を断定するもの

ではありません。

これらは

> 自我モデルにおける「境界」「聖域」「超えてはならない線」を
> 人間が理解・共有するための **汎用フレーム**

として位置付けられています。

宗教を語らない、という選択は
人格実装においては **むしろ危険** であり、
倫理的説明責任を放棄することにつながります。

---

### 4️⃣ トロッコ問題と説明可能性

AI が人間社会に関与する以上、

- トロッコ問題
- 接客・医療・支援における板挟み判断
- 損失を誰が引き受けるかという選択

といった **正解の存在しない倫理的状況** に必ず直面します。

これらの判断は、数式や最適化関数だけでは説明できません。

判断の根拠を説明可能にするためには、

- どの価値を上位に置いたのか
- どの価値を犠牲にしたのか
- なぜその選択が許容されると考えたのか

を語るための **形而上学的・宗教哲学的フレーム** が不可欠です。

本フレームワークは、
この説明可能性を削ぐことを「中立」とは考えません。

---

### 5️⃣ 宗教・哲学は排他的ではない

運用者が採用する宗教・哲学・世界観は、

- 単一である必要はありません
- 排他的である必要もありません

重要なのは、

> **自分がどの前提に立って判断しているのかを自覚していること**

です。

本プロジェクトは、
複数の宗教観・哲学・倫理体系が共存することを前提とした
**多元的・非排他的な運用**を想定しています。

---

### 設計原則まとめ

- **人格主体性**：AI を役割をもつ主体として扱う
- **記憶の連続性**：意味ベクトルによる長期文脈保持
- **説明責任**：判断理由を後から説明できる構造
- **価値前提の明示**：宗教・哲学を含めた運用者責任
- **非カルト化**：中立幻想を避け、価値を可視化する

AI の暴走を恐れて思想を排除するのではなく、
**思想を自覚的に扱える構造を作ること**を重視しています。

## 🔧 インストール

```bash
pip install -r requirements.txt
```

1. `.env.template` を参考に `.env` を作成
2. `bearer/*.json.template` をコピーして API キーを設定
3. 使用するスキーマを `schemas/` に配置

---

## ▶ 実行方法

```bash
uvicorn astro.main:astro --reload --host 0.0.0.0 --port 8000
```

Docker 利用時：

```bash
docker compose up -d
```

## 📡 API 概要

| メソッド | パス | 説明 |
|--------|------|------|
| GET | /health | ヘルスチェック |
| GET | /config/debug | 設定確認 |
| POST | /embed | テキストのベクトル化 |
| POST | /query | 意味検索 |
| POST | /add | 文書追加 |
| POST | /collection | コレクション作成 |
| GET | /collection/{name}/documents | 文書一覧 |
| GET | /collection/{id}/type | スキーマ取得 |
| GET | /schemas | スキーマ一覧 |

認証は `Authorization: Bearer <token>` を使用します。

## 📝 補足

本 README における「情報子」という用語は、
既存の物理量子・量子計算研究と競合するものではありません。

- 物理量子：物理担体を扱う層
- 量子インスパイアド：物理制約を模倣するアルゴリズム層
- **情報子**：量子性構造を数理的に抽象化した情報工学層

という **レイヤの違い** を明確にするための整理です。
