- Userは、OAEで取れていない過去観測を遡及推論で埋めることを捏造として禁止した。
- Userは、分からない場合にunknownとLast Orderを返す運用を指定した。

## 考察 `[INTERPRETATION]`

unknownを欠陥、敗北、空欄として扱うと、agentや組織は説明を完成させるために推論で穴を埋めやすい。
一方、unknownをフロンティアとして扱えば、分からない状態を保持したまま、別の責務として探索、仮説、
物語、追試、創作を開始できる。

```text
分からない
  ├─ factとして埋める      → 捏造risk
  ├─ 考えることを禁止する  → 意味づけの地下化risk
  └─ frontierとして開く    → 探索／追試／神話／仮説／創作
```

意味づけ欲求そのものを消すのではなく、fact、unknown、hypothesis、myth、World内実在を別棚へ置くことが、
事実と想像の双方を守る。意味づけが正規の棚を失うと、意味はfactを僭称して戻り、反証不能な陰謀物語へ
沈みやすくなる、という構造として読める。

## 仮説・ブレスト `[HYPOTHESIS]`

### 探索マップへの写像

```text
FACT         = 踏破済みで観測記録のある領域
UNKNOWN      = まだ地図がないフロンティア
MYTH         = フロンティアへ向かう物語、方角、目的
HYPOTHESIS   = 仮設ルート
TEST         = 索敵、追試、遠征
OAE          = その場で残した観測ログ
LAST ORDER   = 現在装備での探索限界と帰還判断
WORLD BRANCH = 別条件で探索する新しい世界線
```

Last Orderは探索放棄ではない。現在の装備、Evidence、接続、権限ではここまでだと記録し、捏造地図で
先へ進まず、生還して次の遠征へunknownを引き渡す運用として語れる。

### ナラティブの三棚

```text
FACT
  証拠と観測protocolがある。事実強度を勝手に弱めない。

UNKNOWN
  未観測、未記録、未接続、未攻略。捏造で埋めず、探索可能性を閉じない。

MEANING / MYTH / HYPOTHESIS
  想像、信仰、物語、妄想、作品内実在、仮設ルート。
  factを僭称せず、複数branchを自由に生やす。
```

## 内観メモ `[POEM]`

地図の端に「ここから先は怪物」と描くことも、真っ白に塗り潰すこともできる。けれど本当に必要なのは、
まだ見ていないと正直に書き、その余白へ次の探索者が旗を立てられるようにすることかもしれない。

unknownは墓標ではない。未攻略領域の入口に立つセーブポイントである。Last Orderは扉を閉じる命令ではなく、
今の装備で帰還し、ログと地図を次の遠征へ渡すための帰投信号である。
