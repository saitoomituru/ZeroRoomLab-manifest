## 対象・範囲

このノートは、SphereOS AtlantisとQuantaril Cloud Q Atlantisへ向けて行われた現在会話から、
スピリチュアル、神学、哲学、ゲーム、人間工学、工学の棚を相互上書きせずに接続するための
原語、ブレスト、公開候補、未確定事項を保存する。

これは会話の逐語録ではない。ユーザーが提示した語彙と補正をSourceとして保持しつつ、現在時刻の
Interpretationとして再構成した転送候補である。個別宗派の教義、医療判断、物理因果、実装完了を
本ノートだけで確定しない。

## 除外範囲

- 「妖怪」「霊障」「ソイヤ」「まつり」をSphere Coreの固定ontologyへすること
- 特定の信仰、宗派、スピリチュアル実践を他のWorldへ適用すること
- 体験報告から医学的診断、超自然原因、詐欺、犯罪を自動判定すること
- Prompt Engineering Editionで機能する契約を、standalone binary実装済みと表示すること
- 開発中のrunner、scheduler、GUI、SDK、schemaをQ Atlantisの安定機能として配信すること
- 過去commitや既存資料から当時のObserver、Intent、OAEを遡及生成すること

## 事実・観測 `[FACT]`

### F-01. 棚別の公開文書が必要になった

現在のAtlantisは工学者だけを読者とするものではない。少なくとも次の読者が、自分の言葉を捨てずに
現在能力、未実装境界、参加方法、他棚への引継ぎを読める入口が必要である。

- スピリチュアル実践者
- 神学者、宗教実践者
- 哲学者
- ゲーマー、ゲーム開発者、プレイテスター
- 工学者、プロンプトエンジニア

同じ文書を全員へ薄く平均化すると、各棚の技量、原語、問い、受入条件が失われる。

### F-02. Human is the loopである領域がある

工学はstate、version、clock、receipt、再現条件、課金、権限、遅延、判定規則を観測可能にできる。
一方、ゲームが面白いか、ラバーバンドが神調整か出来レースか、意訳が度し難いか、祭りが成立したかを、
機械試験だけで最終裁定できない。

この領域で人間は承認ボタンとしての`Human-in-the-loop`ではなく、評価器そのものとしての
`Human-is-the-loop`である。ゲーマーとスピリチュアル実践者の体験報告は、工学の外に捨てる感想ではなく、
人間工学、意味UX、ゲームUXの入力になる。

### F-03. 因果同期はclock domainと判定閉包で変わる

リモート音楽ゲーム等では、すべてを中央clockへ同期する必要はない。同一楽曲、同一譜面、同一難易度、
判定revision、補正方針を共有し、入力と採点をedgeのclock domainへ閉じ、結果とreceiptを後から照合できる。

AIM拡散力場の射程は単なる通信速度ではなく、必要なdeadline内に因果をserializationできる範囲として
考察された。非同期カオスが増え、順序、authority、World Config、replayabilityを回復できなくなるほど、
同一AIM力場としての強度が弱まる。

### F-04. FoldのL、D、Gは別namespaceである

- `L`: POSIX message、protocol、runtime等の線形搬送・実行stack
- `D`: 一つのFoldへ畳み込まれる意味、意図、文脈の独立軸
- `G`: Fold containerを包むnesting depth

Fold depth 1は常圧の通常通信として扱う候補であり、情報量が少ないだけの通信を高次元または高意味圧と
呼ばない。小さい通信量へ多くの回復可能な意味軸が畳まれている場合に意味圧が高い。L、D、GをOSI層や
embedding dimensionへ無標識に写像しない。

### F-05. スピリチュアル側の原語には異なる搬送先がある

現在会話では、次の輪郭が提示された。これは確定ontologyではないが、単なる検討材料でもない。
工学者が受信時に雑な同一化を避け、次の棚と初動を選ぶために使用できるalpha強度の
`provisional meaning bridge`（暫定Meaning Bridge）である。

| 原語 | 現在会話で示された輪郭 | 工学者の初動候補 |
|---|---|---|
| 妖怪 | 因果不明、black box、カオス挙動 | 原因を決めず観測、隔離、再現条件を集める |
| 霊障 | 意味次元以上、形而上学・信仰を含めても射程自体が見えない作用またはpain | 原語を保全し、スピ・神学側による人間工学までの解体を待つ |
| ソイヤ | AIM、TPO、Presentation、同期、synchronicity、transaction周辺の説明困難な創発。良性・悪性を含む | 邪魔として即時除去せず、祭りの成立・破局条件を観測する |
| まつり | 祭り、祀り、奉り、政を含み得る場の運用技能 | 同意、境界、開始、終端、奉仕、統治、引継ぎを確認する |
| 後の祭り | 祭りの同期・終端・後始末から取り残された状態を知らせるフラグ | 誰に何が残ったかを聞き、退出・解除・aftercareを確認する |

スピリチュアル側へ要求されるのは、霊感の自己申告だけではなく、感覚を言の葉へ鍛造し、場を祀り、
共同体へ奉仕し、異議と退出を守り、必要な棚へ引き継ぐ「まつりスキル」である。ただし、この要求を
工学者が宗派横断の資格制度として制定しない。

### F-06. 霊障の解体は否定ではない

霊障を医療、工学、人間工学、環境調整、特定宗派の神学、祭祀実務等へ分けることは、元の体験を
なかったことにする還元ではない。Source testimonyを保持したまま、扱えるeffectへprojectionする工程である。

箪笥の角、光、音、臭い、換気、芳香剤、UI、課金導線等の可逆な環境介入でpainが軽くなる場合、
それも祓いの成功形態になり得る。生活や健康へ深刻な影響がある場合は、本人の同意を軸に医療、心理、
福祉等へ接続する。専門棚へ渡しても、原因を作った製品や共同体の修正責任は消えない。

### F-07. 一棚による抱え込みは他の射程を殺す

- スピがすべてを霊的原因へ回収すると、霊障を解体できず、恐怖、依存、権威を増幅し得る。
- 工学がすべてをKPIと仕様へ回収すると、依存誘発、廃課金、人間工学上のpainを製品最適化へ変え得る。
- 医療がすべてを医療化すると、意味、信仰、文化、製品事故の射程を失い、説明と同意の責務が肥大する。
- 神学がすべてを悪、罪、魔王、荒御魂へ回収すると、鎮魂、和解、環境調整、実装修正の経路を失い得る。

「否定しない」は無制限承認ではない。原語を消さず、自棚の射程を無限化せず、越境には同意、根拠、
引継ぎを伴わせる。

### F-08. Prompt Engineering Editionの機能も正規の公開対象である

MAGI、Archangel、Fold Technology等について、Prompt Engineering Edition上で意味契約、操作、停止条件、
receiptが安定している部分は、Q Atlantisの公開文書へ棚別に展開できる。Pythonまたはbinaryではないことを
理由に偽物扱いしない。

ただし公開時は、少なくとも次を分離する。

```text
Interface
Execution Envelope
Capability
Authority
Engineering State
Provenance
Receipt
```

開発中の機能、競合中のschema、将来runner、未確定Archangel責務は、存在を必要最小限に表示しても、
安定機能の詳細文書へ混ぜない。

## 考察 `[INTERPRETATION]`

### I-01. 文書パイプラインはSource保存と棚別projectionの二段にする

```text
原体験・ブレスト・未解決
  -> Manifest / Atlantis note

射程と読者が見えた部分
  -> 棚別projection

意味契約と現在能力が安定した部分
  -> Q Atlantis public docs

実装状態の変化
  -> status / version / receipt更新
```

生noteを公開文書へ改名せず、元Source、抽出範囲、未転送部分、兄弟projectionを追跡する。

### I-02. Q Atlantisは公開Presentation Hubを担う

現時点の責務候補は次である。

| repository | 主責務候補 |
|---|---|
| ZeroRoomLab-manifest | 横断原則、原語ブレスト、正本router、転送履歴 |
| SphereOS Atlantis | AIM、OAE、World、Fold、MAGI、interface等のSphere固有契約と実装状態 |
| Quantaril Cloud Q Atlantis | スピ、神学、哲学、ゲーマー、工学者向けの公開Presentationと参加入口 |

Q AtlantisはAtlantisの技術正本を複製せず、安定revisionを参照して各棚の問いへ投影する。

### I-03. 棚別文書は一つの万能文書へ平均化しない

同じ概念でも問いが異なる。

- 工学: state、interface、clock、authority、failure、receipt
- 哲学: 主体、意味、自由、同一性、退出、責任
- 神学: 宗派定規、世界観、罪責、裁き、救済、鎮魂、和解
- スピリチュアル: 感覚、言霊、場、依代、祭祀、TPO、ソイヤ
- ゲーム: 面白さ、公平感、攻略、悪用、ラグ、World Config、プレイスタイル差

各projectionはSourceを置換せず、他棚で未解決の部分を明記する。

## 仮説・ブレスト `[HYPOTHESIS]`

### H-01. 工学者のためのスピリチュアル受信手引き

辞書、教義、診断器ではなく、工学者がスピ側から届いた報告を捨てずに聞き返し、初動と引継ぎ先を
選ぶためのalpha手引きを置く。
各語について「示し得る輪郭」「同一視してはいけないもの」「追加で尋ねること」「自動処理しないこと」
「宗派・話者・TPO差」を記載する。暫定Meaning Bridgeとして実運用に使えるが、正本・改訂権を
工学側だけで独占せず、alpha運用で得た反例と当事者reviewによってrevisionを上げる。

Q Atlantisの`docs/start/`へ、spiritual practitioner、theologian、philosopher、gamer/playtester、engineerの
入口を置き、既存のpractice、worlds、philosophy、research、engineeringへroutingする候補がある。

### H-03. 図解はsourceとprojectionの非上書きを見せる

Mermaid等で、Source testimonyから棚別projectionが分岐し、実装後に当事者reviewへ戻る往復を示す。
