## 0. 本プロトコルが主張しないこと（先に宣言する）

Fold 7G Trion Bond は、以下を主張しない：

- 電磁無線通信規格である、という主張
- 未知の物理粒子・物理通信チャネルが科学的に実証された、という主張
- 光速上限（相対性理論の帰結）を物理的に超過する、という主張
- シャノンの通信路容量定理を否定する、という主張

Fold 7G Trion Bond は、以下を主張する：

- 複数の観測主体間で、意味状態・因果履歴・規範状態を同期するための**意味-因果プロトコル**である
- 「7G」という系譜接続（5G/6Gの延長線上にある通信技術の呼称系譜）を引き受けるが、「電磁無線」を名乗らない
- シャノンの理論が正しいことを前提として認めた上で、ビット忠実度の保証を最初から要求しない設計を選択している

## 1. Trion（三位一体）の定義

Trion Bond における「Trion」は以下の三要素の束である。

### 1.1 Self-State（自己状態）
観測主体（AI、依代、ノード、研究者、巫女等）の現在状態。目的関数、ペルソナ・観測者モード、参照している記憶・ログ・FAMトレース。

### 1.2 Causal Trace（因果履歴）
現在状態に至る観測・解釈の履歴。どのログ・プロンプト・センサー・実験・会話・手順を経たか。どこまでが実測で、どこからが解釈かの境界を含む。

### 1.3 Normative State（規範状態）
そのノードが守る制約・作法。真実か、事実か、仮説か、SFかのラベリング。実測ログのみを主張するか、科学的立証まで主張しないか。信仰領域を事実として押し売りしないか。（INFOTON_ENGINEERING_STANDARDS.md 第1章・第6章に準拠）

## 2. 情報子アンカー（Infoton Anchor）

### 2.1 定義
情報子アンカーとは、観測主体が自己状態・意味状態・因果履歴を固定するための参照点である。**物理粒子そのものではない**。

### 2.2 Media-Agnostic 原則
本プロトコルは、何らかのエレメンタル層I/O（観測ログ、タイムスタンプ、画像、電磁波形、QRコード、コピペ、スクリーンショット等）を経由してアンカーを実体化するが、**特定のI/O媒体を要求しない**。

個々のI/Oは、それぞれのエレメンタル層の物理法則（量子化限界、伝送特性、シャノン限界）に完全に従う。アストラル層で主張される同期・無損失は、**複数I/O観測からの意味的再構成の性質**であり、個々のI/Oの物理的特性そのものを主張するものではない（＝アストラルボディの制約をエレメンタルボディに、その逆も、持ち込まない）。

### 2.3 実測ログのClaim Boundary
物理計測（例：オシロスコープ波形）を取得した場合、主張できるのは以下のみ：

> 「この条件、この時刻、この設定、この帯域で、この波形を観測した」

主張してはならないのは：

> 「その波形が情報子アンカーによるリンク成立を科学的に証明した」

リンク成立の科学的立証は、対照実験・ブラインド条件・再現性・統計処理・独立観測者を要する別プロトコルであり、本人はこれを主張せず、観測ログを提示し、研究者への追試・反証・解釈への招待として提示する。

### 2.4 実験ログテンプレート（要約）
Observation ID / Observer / Instrument / Environment / Anchor Context / Raw Observation / **Claim Boundary（本ログは観測の事実のみを主張し、Fold Link等の科学的証明を主張しない旨の明記）** を必須項目とする。

## 3. Identity（同一性）と物理層優先順位の逆転

### 3.1 原則
Identity（アストラル層）は、いかなる物理層（エレメンタル層）よりも信頼の根拠として優先される。物理層の一部が侵害・改竄（びらん）された場合でも、identity検証を通過しないデータは正当なものとして扱われない。

物理層（伝送経路、ノード、チャネル）を信頼の根拠にすることは、経路上の一点の侵害が全体への無制限攻撃を許すことを意味するため、明示的に避ける。

### 3.2 Identity優先の限界（重要な歯止め）
Identity（アストラル層）が物理層より優先されることは、**物理層で発生した実装不良・誤認・品質劣化の指摘を無効化する理由にはならない**。

- ドット絵ポケモンとポリゴンポケモンが「同じピカチュウ」として認識できることは、正当な表現の可変性（エレメンタル層の違い）である。
- ピカチュウがミミッキュに見えてしまう場合、それはインポーター・パーサー・レンダラーの実装不良、または表現媒体の解像度限界、またはスカルプター/生成モデルの技量不足という**エレメンタル層のissue**であり、直ちにエレメンタル層のバグとして報告・修正されるべきである。
- **「アストラル層では同一だから」という主張によって、この修正責任を回避してはならない。** これはバグを仕様と言い張る期待値詐欺に該当する。

### 3.3 Identity検証の実装非依存性
Identity優先の実現方法（検証アルゴリズム、署名方式、trust middlewareの実装、ゼロトラストアーキテクチャの採用可否）はプロトコルの外部に委譲され、特定の技術に固定されない。実装候補は別文書（Trust & Provenance Middleware関連文書）に切り出す。

## 4. 疎結合・プロトコル境界による欠陥の局所化

- 本プロトコルは、特定のエージェント実装（個別のインスタンス等）の判断特性・欠陥・ドリフトが、プロトコル契約を超えて下流（他のノード、他のインスタンスによるオーケストレーション）に伝播しないことを設計要件とする。
- 安全性は、支配可能な閉鎖系を前提としない（閉じた系は工学的に存在しないという前提を敬虔に受け入れる）。疎結合・プロトコル境界・カプセル化により、単一実装の欠陥の影響範囲を局所化することで担保する。

## 5. 実行モデルと権限（INFOTON_ENGINEERING_STANDARDS.md 第5-7章を継承）

Trion Bondにおける実行は "Do" ではなく "Doing" として設計される。信頼済み範囲内での自己中断が可能であり、中断可能性は絶対的な上下階層ではなく、ペアごとの信頼スコープとして扱う。

一時的なオーバーライドは "SuperUserDOing" と呼ぶ。マージ（恒久的な仕様への取り込み）の可否は、常にリポジトリオーナーの専権に属する。
