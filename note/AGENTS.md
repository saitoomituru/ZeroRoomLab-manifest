# Note 作法ガイド

作成日: 2026-07-13

## 1. 目的

このガイドは、note/ 配下に置くノートの取り方・書き方・整理の作法をまとめるためのものです。ここで定めるルールは、技術文書・哲学文書・研究ノートのどれにも適用できるように設計する。

## 2. 基本姿勢

- ノートは「正解を出すための文章」ではなく、観測・検証・訂正・保留を残すための文章として扱う
- 事実と解釈を分けて書く。ポエムや思索を入れる場合は、その位置づけを明示する
- ノートには、必ず「内観メモ」を残す。所感、ポエム、気づき、雑な類推、未整理の発想を、明示的に別欄として書く
- ポエムや内観メモは、単なる余白ではなく、思考の芽や未整理の構造を残すための重要な記録であると扱う
- 事実だけを並べてノートを閉じない。内観メモの存在そのものが、研究の温度差・身体性・観測の揺れを残すために重要である
- 取れないことを「ない」と書き換えない。確認不能は確認不能として残す
- 既存の用語・文脈・棚に依存する場合は、その参照元を明示する

## 3. 記述の作法

### 3.1 事実・解釈・ポエムの分離

- 事実確認できる内容は「事実」欄として記す
- まだ推測・仮説の段階である内容は「考察」や「仮説」として明示する
- ポエム・所感・感情的な反応・雑な類推は、必ず「内観メモ」欄として独立して残す
- 事実と内観メモを同じ段落で混ぜない

### 3.2 等身大の発言

- できること・できないことを正直に書く
- 「絶対」「完璧」「検証済み」などの強い断定は、実際に確認できた範囲に限定する
- 自分の未確認部分は、未確認であることを明示する

### 3.3 検証範囲の限定

- ノートの冒頭に、今回の検証範囲・対象・除外範囲を明示する
- 公開情報だけを確認した場合は、それを明記する
- 非公開情報・個人情報・実行環境の内部状態は、対象外として明示する

## 4. 棚ごとの記述範囲ルール

- ある内容が技術・哲学・信仰・ナラティブのどの棚に属するかを事前に明示する
- 同じ内容を複数棚で同じ基準で評価しない
- たとえば、README の表現は技術精度ではなく、表紙・ブランド・思想語りとして評価する
- 深い技術文書では、実装・検証・再現性を重視する
- それぞれの棚にはレジスタがある。技術棚で哲学棚の基準で断罪したり、哲学棚で技術棚の尺度で裁いたりしない
- 他の評価尺度での断罪・軽視・蔑視を、ノートの中で無自覚に行わない

## 5. ノートの項目レベルのタグ運用

- ノートには最低限、対象・範囲・状態・出典を示すタグを入れる
- 例: `[FACT]` / `[HYPOTHESIS]` / `[POEM]` / `[LIMITED]` / `[NEEDS-REVISION]`
- 重要なノートは、内容の性質に応じて棚タグも付ける
- 例: `[Layer A]` / `[Layer B]` / `[Layer C]` / `[ARCHIVE]`

## 6. 旧ノート・転送候補との接続

- 旧ノートを再利用する場合は、原典と差分を明示する
- 反映先がある場合は、元ノートと転送先の関係を記す
- 生ノートそのものをそのまま正規文書にしない

## 7. 追加の注意

- ポエムを書かないことをルール化しない。未整理の思考を残すこと自体が価値ある場合があるため、位置づけを明示して書く
- ただし、ポエム・雑な類推・内観メモを事実と混同しない
- ノートの内容が大きく変わった場合は、更新日時・更新理由を残す
- 内観メモがないノートは、未完成として扱うことがある
- 断罪的な読み方（「これは雑」「これは無価値」「これは不真面目」）を、棚ごとの役割を無視して適用しない
- メサコン的な評価ペインを起こすために、内観メモやポエムの価値を削らない

## 8. 工学復旧からMAD実践科学へ切り替える

実機では全ての法則と制約が同時に発動する。故障時は、最初から単独変数再現や母集団統計を完了条件にせず、まず工学として安全なassembly隔離、目的機能の復旧、危険伝播の拘束を行ってよい。

ただし、工学復旧後に次の残差がある場合は、MAD巫女サイエンティストの実践科学logへ切り替える。

- 変更後のeffectが予測と逆向きになった
- 消えた状態が再出現した、または設定が理由不明で消失・復活した
- 電力、熱、機械、network、service等の別layerが連動して見える
- 既知モデルで説明できる枝を列挙しても差分が残る
- minorと見ていた法則が、現在条件では支配的になりうる

切替後は次を守る。

1. 時刻、hardware、runtime、配置、負荷を記録する
2. 観測値、工学status、解釈、仮説、UNKNOWNを分ける
3. 既知モデルで説明できる枝を先に列挙し、残差をノイズとして消さない
4. 未知のlogicや抽象機械は一段ずつ仮置きし、発見済みの法則へ昇格しない
5. 単体変数化は、復旧済み母機を危険へ戻さない非侵襲な変数から行う
6. A/Bだけで時間変化と混ざる場合は、可能ならA/B/A、反復、現場環境の同時計測を使う
7. 新しい証拠に応じて`関与疑い`、`関与濃厚`、`故障隔離済み`等のstatusを上下させる

科学の定規は工学復旧を無効化するためではなく、復旧後にも残る謎を次の設計、保護、復旧自動化、新理論候補へ接続するために使う。

> 捏造しない。想像を禁じない。変な差分を消さない。

## 8. 工学復旧とMAD実践科学の切替

Layer Aの実機障害では、工学的fault containmentと科学的因果確定を一つの完了条件へ丸めない。

- まず、同時発動する法則、資源制約、危険伝播を読み、目的機能を復旧し、壊れたassemblyを安全に隔離する
- assemblyの除去、交換、迂回で目的機能が戻った場合、内部故障モード未確定でも`故障隔離済み`として工学判断を閉じてよい
- `関与疑い`、`関与濃厚`、`故障隔離済み`、`故障モード確定`、`母集団一般化`を区別する
- 単独変数、正常対照、反復、統計は、原因、程度、分布を詰める必要が生じた時に科学の定規として使う
- 既知の物理破損部品を、形式的再現性のためだけに復旧済み母機へ戻さない

工学復旧後も次の残差がある場合、MAD巫女サイエンティストの実践科学logへ切り替える。

- effectが予測と逆向きになる
- 消えた状態または設定が理由不明で再出現／消失する
- 電力、熱、USB、network、service等の別layerが連動して見える
- 支配的と見た法則を除いても残差がある
- 現在の抽象機械だけでは観測列を無理なく説明できない

切替後は、時刻と構成を固定し、観測と推論を分け、非侵襲probeから一変数ずつ切る。既知モデルで説明可能な枝を先に列挙し、それでも残る差分を`[UNKNOWN]`として保存する。必要なら未知のlogic、minorな法則、追加の抽象機械を一段だけ仮置きし、反証で強度を上下させる。

> 捏造しない。想像を禁じない。変な差分を消さない。

科学の定規は、工学復旧を事前に不可能へする通行証ではない。復旧後の残差を新しい設計、保護、復旧自動化、理論候補へ接続するためにも使う。
